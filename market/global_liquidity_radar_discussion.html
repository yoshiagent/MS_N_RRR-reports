④ codex 顧問裁決（GPT-5.5，含 web search）— 摘要

議題codex 觀點

📜 codex 顧問全文（原文，點開展）
以下用批判角度講結論：這套「全球流動性風險雷達」可以做，但不應該太快接進 regime。它比較像 風控 overlay / kill-switch 候選因子，不是穩定的 regime 判斷核心。最有價值的是「美股半導體隔夜訊號」和「日圓急升造成的槓桿平倉壓力」，但很多訊號會跟你現有的 VIX + 外資期貨 OI + TWII MA 高度重疊。

────────────────────────────────────────────
1. 這些跨市場訊號對台股是領先、同步、還是落後？

先分清楚「經濟上領先」和「交易上領先」。台股交易時間在亞洲盤，美股 SOX/NVDA 是前一晚收盤資訊，所以對台股隔日開盤有交易上的領先性；但 KOSPI/KRW 多半和台股同屬亞洲盤風險資產，很多時候只是同步反應。

● JPY carry unwind：有時真領先，但更多時候是尾部事件偵測器。
  日圓急升、USD/JPY 快速下跌，確實可能代表 carry trade 平倉、槓桿降風險、全球資產同步賣壓。
  研究與市場經驗都支持 carry trade 存在尾部風險與跨市場風險傳染，尤其在高槓桿、波動率升高、利差反轉時更明顯。
  但陷阱是：你看到 USD/JPY 暴力升值時，通常 VIX、日股、美股期貨、亞幣、外資期貨部位也已經在反應。
  它未必比 VIX 或外資期貨 OI 更早。真正有用的不是「日圓升值」本身，而是
  日圓急升 + 全球股市下跌 + 波動率跳升 + 利差壓縮 這種組合。
  判斷：偶爾領先，常常同步。適合當 crash-risk detector，不適合當一般 regime 因子。

● KOSPI 破位：對台股多半同步，不是穩定領先。
  韓股與台股都高度連動全球電子、記憶體、半導體循環。KOSPI/KOSDAQ 破位通常反映的是
  同一批全球資金在亞洲科技鏈降風險。日線級別上，它對 TWII 不一定有穩定領先性。
  交易上，韓股開盤比台股早約一小時，所以盤中第一小時 KOSPI/KOSDAQ 跌破關鍵位可能對台股當日有一點訊號價值。
  但如果你只用 Yahoo 日線收盤資料，那韓股收盤其實晚於台股收盤，不能拿當天 KOSPI 收盤去判斷當天台股，否則是前視。
  判斷：日線收盤多半同步或落後；盤中早盤資料才可能小幅領先。

● KRW 走弱：比 KOSPI 稍有機會領先，但容易跟 USD/TWD 重複。
  韓元通常比台幣更市場化、更敏感，對亞洲出口鏈、美元流動性、外資風險偏好反應快。
  KRW 走弱有時會比 TWD 早釋放壓力，因此可作為亞洲風險偏好的 proxy。
  但你已經有 USDTWD，再加 USDKRW 可能只是多一個高度相關的亞洲 FX risk-off 指標。
  它的增量價值要看：KRW 是否在 TWD 尚未貶、TWII 尚未破線前，就能預警後續台股 5-20 日下跌。
  判斷：可能小幅領先 USDTWD/TWII，但需要嚴格驗證。單獨用容易變成重複資訊。

● SOX/NVDA 破底：交易上最明確領先台股，但不一定比台積電 ADR / Nasdaq / 費半期貨更好。
  這是四個裡面最合理的。台股權重高度集中在半導體，SOX/NVDA 前一晚大跌或破底，對隔日台股尤其電子權值股有直接影響。
  這不是宏觀先行，而是時間帶來的隔夜領先。
  但問題是：如果你的現有 market agent 已經透過 TWII MA、VIX、外資期貨 OI 反映到台股內部趨勢，
  SOX/NVDA 的價值主要在更早一日或更早一個開盤捕捉電子股壓力。它比較適合作為隔日開盤降風險或選股扣分，
  而不是重新定義大盤 regime。
  判斷：對台股隔日最有交易領先性。建議優先測。

────────────────────────────────────────────
2. 用「是否領先 TWII 回檔」回測，最容易犯哪些錯？怎麼設計才穩健？

最大錯誤是把「已經知道 TWII 回檔」拿來反推訊號。常見陷阱：

1) 前視偏誤
   例如用當天 KOSPI 收盤、KRW 紐約收盤、SOX 收盤去預測同一天 TWII 表現，時間順序很容易錯。
   正確做法是所有特徵都要有 as_of_time。台股 09:00 開盤前，只能用前一晚美股收盤、前一交易日 KOSPI 收盤、
   當時可觀測匯率、期貨夜盤資料。不能混用不同市場的「日期」。

2) 用未來定義破位
   例如「SOX 破前低」如果前低是用整段資料滾動最低，會偷看未來。
   正確做法是只用當下之前 N 日資料，例如 close < rolling_min(close, 60).shift(1)。

3) 事件定義事後最佳化
   160、破 MA60、三日升值 3%、五日跌 5%，這些門檻如果是看完歷史才挑，就是資料窺探。
   正確做法是先定義少量合理規則，再做 walk-forward。門檻選擇只能在訓練窗內完成。

4) 樣本重疊造成假顯著
   如果每天都測「未來 20 日是否回檔」，相鄰樣本高度重疊，t-stat 會膨脹。
   正確做法是用事件法、purged walk-forward、block bootstrap，或者用 non-overlapping windows 補驗。

5) regime 循環論證
   你已經證明動能溢價是 regime-conditional，現在又用 TWII 回檔定義新訊號價值，
   最後可能只是證明「下跌前會有下跌跡象」。
   正確做法是把評估目標改成策略層面的增量：加入訊號後，是否提高 out-of-sample CAGR、降低 MDD、
   改善 downside capture、減少大虧換股，而不是只問能不能解釋 TWII 回檔。

6) 忽略交易成本與可執行性
   risk-off 訊號如果觸發後隔天開盤已經 gap down，降低曝險可能賣在最低點。
   正確做法要測：訊號在台股開盤前已知時，以開盤、VWAP、收盤執行各自結果如何。

穩健設計建議：
 - 每個特徵建立嚴格的 available_at 時間。
 - 先做「觀察頁」是對的，但觀察頁也要記錄訊號快照，不能事後重算。
 - 先測單一訊號的 forward return distribution，再測加入現有 regime 後的 incremental value。
 - 用 walk-forward：例如 2010-2018 訓練、2019-2021 驗證、2022-2026 測試，或 rolling 5Y train / 1Y test。
 - 評估不只看勝率，要看 tail：觸發後未來 5/10/20 日 TWII 的左尾、最大不利變動、個股策略 drawdown。
 - 和 baseline 比：現有 regime vs 現有 regime + 新 risk-off score，不要只測新訊號本身。

────────────────────────────────────────────
3. risk-off 訊號一年沒幾次（樣本少），怎麼證明有增量價值？需要多少事件數？

先講難聽的：一年觸發 3-5 次，台股完整可用歷史就算 15 年，也只有 45-75 次，而且這些事件不獨立。
真正獨立的全球風險事件可能不到 20 次。你很難用傳統統計「證明」。

事件太少時，不能追求漂亮 p-value，要追求三件事：
1) 方向一致性：觸發後未來 5/10/20 日的平均報酬、左尾風險、策略回撤，是否在不同年代都同方向改善。
2) 經濟意義：即使事件少，只要能明顯降低 2008、2011、2015、2018、2020、2022、2024 類型的 drawdown，
   對策略可能仍有價值。重點是有沒有改善 MDD、Calmar、left-tail loss，而不是勝率。
3) 增量，不是替代：要測在控制 VIX、TWII MA、外資期貨 OI、USDTWD 後，新訊號是否還有解釋力。
   可以用 logistic / hazard model / tree-based model，但要限制自由度。

事件數粗估：
 - 少於 20 個獨立事件：只能當風控規則或研究假說，不能說有統計證明。
 - 30-50 個獨立事件：可以初步判斷方向，但信心仍低。
 - 50-100 個獨立事件：才比較能評估訊號穩定性。
 - 100+ 個事件：才適合做較細的門檻、分層、交互作用。

但 risk-off 本來就是尾部事件，不太可能有 100 個乾淨樣本。比較務實的方法是把訊號做成連續風險分數，
例如 JPY 5 日 z-score、KRW 20 日動能、SOX 距離 60 日低點、VIX term shock，而不是二元觸發。
這樣樣本變多，但要用 HAC / block bootstrap 處理自相關。

────────────────────────────────────────────
4. 「160 定律」這種固定價位門檻，可靠替代是什麼？

固定價位最危險。USD/JPY 160 在某個利率環境、央行口徑、市場倉位下有意義，但不是永恆門檻。
2024 的 160 和 2026、2028 的 160 不一定代表同樣壓力。
比較可靠的 carry unwind proxy 應該是「變化率、波動率、利差、倉位」組合。

1) JPY 升值速度（比固定價位更重要）
   USDJPY 5日報酬 < -2%、USDJPY 10日報酬 < -4%、USDJPY return z-score < -2。
   對 carry unwind，暴力升值比匯率在 160 還重要。
2) 匯率波動率跳升
   USDJPY 10日 realized vol / 60日 realized vol > 1.8；USDJPY ATR percentile > 90%。
3) 利差壓縮
   美國 2Y - 日本 2Y、美國 10Y - 日本 10Y、利差 20 日變化、Fed/BOJ 預期變化。
   若 USDJPY 下跌 + 美日利差壓縮 同時發生，比單純跌破某價位可信。
4) JPY funding stress basket
   不要只看 USD/JPY。看 JPY 是否對多數高息或風險貨幣同步升值：AUDJPY、NZDJPY、MXNJPY、KRWJPY、TWDJPY。
   若 JPY 對整個 carry basket 急升，才比較像去槓桿，不只是美元因素。
5) 交叉資產確認
   Carry unwind 應該伴隨：VIX 上升、Nikkei/Topix 下跌、SOX/Nasdaq 下跌、高 beta FX 走弱、美債殖利率下行。
   單一 USD/JPY 訊號不夠，最好設計成 composite score。

簡化版本：
   JPY unwind score =
     z(-USDJPY 5D return)
   + z(USDJPY realized vol 10D / 60D)
   + z(-(US2Y - JP2Y) 20D change)
   + z(-AUDJPY 5D return)
   + z(VIX 5D change)
當 score 進入歷史 90 或 95 分位，再當作 risk-off，而不是 USD/JPY 到 160 就觸發。

5. 值不值得做？相對現有訊號是否只是重複資訊？明確取捨

我的取捨：值得做觀察頁與研究，不建議短期直接接 regime 總開關。

原因很直接：你現有系統已經有 TWII MA20/MA60（本地趨勢）、OTCI（中小型風險偏好）、
USDTWD（台幣資金壓力）、VIX（全球風險）、外資台指期 OI（外資槓桿方向）、三大法人現貨（本地資金流）。
這些已經覆蓋大部分 risk-off 資訊。新增 JPY/KOSPI/KRW/SOX 很可能有 collinearity。
尤其 KRW、KOSPI、VIX、SOX 在壓力時通常一起動，最後模型可能只是把同一件事算四次，導致 regime 過度轉空、錯過反彈。

但有兩個方向仍值得做：
● 第一優先：SOX/NVDA 隔夜半導體風險。
  這對台股最有交易時差價值。建議不要先接 regime，而是接到：電子股選股扣分、半導體曝險上限、
  隔日換股節奏、高 beta 動能股降權。它是產業鏈訊號，不只是宏觀 risk-off。
● 第二優先：JPY carry unwind composite。
  不要用 160。做成連續分數，專門處理尾部風險。它不應該每天影響 regime，而應該在極端分位時啟動：
  暫停加碼、降低持股上限、提高現金、禁止買進高 beta / 高融資 / 高波動標的。
● KOSPI/KRW：先放觀察頁，不急著接線。
  KOSPI 日線破位多半同步。KRW 可能有一點亞洲 FX 領先性，但很可能和 USDTWD 重複。
  除非回測證明它在控制 USDTWD 後仍能改善策略左尾，否則不值得增加模型複雜度。

明確建議：
 1. 做觀察頁：值得。
 2. 做回測研究：值得，但要用嚴格 as-of time。
 3. 直接接 regime：不建議。
 4. 優先接策略 overlay，而不是總開關。
 5. 若只能選一個做：先做 SOX/NVDA 隔夜訊號。
 6. 若做第二個：做 JPY carry unwind composite，不要做 160 固定門檻。
 7. KOSPI/KRW 先列入監控，不要讓它們左右 regime，除非通過增量測試。

最終結論：這個提案方向合理，但原始版本太像「事後解釋型全球風險清單」。要把它變成可交易的
market agent 模組，必須從「跨市場故事」改成「有時間戳、可執行、控制既有因子後仍有增量」的風控 overlay。
否則只是把 VIX、USDTWD、TWII MA 已經告訴你的事情，用更多市場名稱重複一次。
⑤ 落地路徑（分階段，與 codex 收斂）
報告方向對（補系統最弱的跨市場維度），但不能照抄敘事接線。落地應該是：
階段動作風險/接線程度
1 資料層yfinance 補 JPY=X / ^KS11 / KRW=X（+已有 ^SOX）零風險、不接決策
2 觀察頁 shadow做一個 global_stress_score（波動調整、非四個孤立紅燈）+ 每日訊號快照留痕只顯示、不影響選股/切檔
3 回測驗證嚴格 as_of_time；Base regime vs Base+overlay 的 OOS 增量（MaxDD/左尾/Calmar）研究，不接線
4a 接策略 overlaySOX/NVDA→電子股扣分/半導體曝險上限；JPY composite→極端分位降槓桿接『個股/曝險』，非 regime
4b regime（最後）僅在 OOS 證明控制既有因子後仍有增量，才接 regime，且只當降檔煞車最高門檻、可能永不接
若測完發現只降波動卻砍掉太多報酬 → 永遠留在 dashboard 當人工風險提示，不接決策。此路徑符合家規：重要決策先過 codex、先累積再改、FS 不重算。
🎯 決策要點　真正的價值不是「多抓幾個市場」，而是一個不重複、不偷看、能解釋左尾的 stress overlay。跨市場訊號適合當煞車輔助、不適合搶方向盤。優先序：SOX/NVDA 隔夜訊號（最有交易時差價值）＞ JPY carry composite（尾部風控）＞ KOSPI/KRW（先監控）。此案適合「低成本做研究+監控」，不適合一開始就工程化進核心決策邏輯。
