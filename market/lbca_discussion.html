② 三方收斂（codex 與 Gemini 一致處）
議題共識結論
核心定位❌「低位＋籌碼強」太粗＝主力榜/法人池的低位換皮版 → ✅ 必須是「低位止跌後、籌碼由弱轉強、且尚未被既有榜單捕捉」的早期段
低基期口徑❌ 52週range位置最危險（破底股天然落低段、range 隨新低下移＝假穩定）→ ✅ 改以「過去一年報酬率全市場落後段（<30%）」為主
止跌條件❌ 別只用站上 MA60（落後、易被假突破騙）→ ✅ 加「不再破底」硬條件（近60日未創120日新低／20日低點抬高）
加速平滑❌ 別對日頻直接做二階微分（滿是假陽性）→ ✅ 先平滑（EMA/MA）＋z-score 動態門檻＋多視窗階梯，追「改善速度變快」而非真二階
重疊風險⚠️ 極高機率跟主力榜/inst_entry_pool/認養榜重疊 → ✅ 必須專門做「獨立性設計」，否則只是換皮
回測方法❌ 別只比「交叉 vs 單一 filter」（串聯本就提升品質）→ ✅ 2×2 factorial ＋交互項回歸（看 β3 是否顯著>0）
接線紀律✅ 先 shadow 回測證明有交互效應＋含成本/滑價/漲停買不到/空頭分段，才建池、才升 Tier-2
③ 最關鍵：四招確保「獨立性」（否則就是換皮）
兩位顧問都指出 LBCA 最大的死法是變成現有榜單的低位子集。合起來有四道獨立性防線：
招式來源做法
EarlyStage 排除codex事件日前 20 日『未進』主力進場榜 Top N／inst_entry_pool／投信認養榜；且前 20 日漲幅未超市場前 15%。抓「還沒被發現」的段
去機構化（在地分點）GeminiB 條件『排除三大法人買超』，專注在地分點/囤貨分點的買賣家數差集中。三大法人還沒買、但關鍵少數分點悄悄加速＝真早期璞玉
中小市值分流Gemini限市值 20~80 億——法人受流動性限制買不動、但主力分點能點火，與偏好大型股的 inst_entry_pool 完美互補、天然不重疊
殘差獨立性測試codexLBCA_score 對「主力score＋inst_entry＋SMI_phase＋流動性＋動能」回歸，看 residual 是否還能預測 forward return；沒 edge＝換皮
codex 補一個驗證心法：真早期訊號應看到「LBCA 先出現 → 之後才進主力榜/法人池 → 價格跟上」。若事件後沒有更強籌碼訊號接力、只是價格短彈，那它是反彈交易、不是早期吸貨模型。
④ 收斂後的可實作 v1 設計（待 CIO 拍板門檻數字）
LBCA = LowBase_Safe ∧ ChipAccel_Smoothed ∧ EarlyStage ∧ Tradable
模組條件（codex 主幹＋Gemini 補強）
LowBase_Safe（低基期＋安全）① 過去一年報酬率全市場分位 < 30%（主條件·抓「相對沒漲到」）｜② 距 52 週高點乖離 > 25%（補空間感）｜③ 近 60 日未創 120 日新低（防破底）｜④ MA20 斜率轉正、站上 MA60（止跌）｜Gemini 補：或 近 20 日布林帶寬壓縮至近一年新低（籌碼沉澱）｜52週位置降級為輔助（20~60%，不低於20%）
ChipAccel_Smoothed（平滑後加速）① CMG_MA20_z60 > 0｜② Δ(CMG_MA20_z60, 5日) > 0｜③ 多視窗階梯 CMG_5D_z60 > CMG_20D_z60 > CMG_60D_z120｜④ 5 日內至少 3 日主力買方家數差為正｜Gemini 補：加速當日量 > 5 日均量 1.5 倍（防對敲）、z-score > 1.5 才算顯著
EarlyStage（尚未擁擠·獨立性）前 20 日未進 主力榜TopN／inst_entry_pool／認養榜｜前 20 日漲幅未超市場前 15%｜Gemini 補：B 條件去機構化（排三大法人、專注在地分點）＋限市值 20~80 億中小型
Tradable（可交易）排除處置股／全額交割／流動性不足／連續漲停鎖死；注意股單獨分層
codex 提醒拆兩版對照：LBCA_raw（不排既有榜單）vs LBCA_ex_legacy（排除近期已入榜）。真正有價值的是 LBCA_ex_legacy；若只有 raw 有效、ex_legacy 無效，就證明它只是換皮。
⑤ 回測方案：證明「交叉 > 單條件之和」（有 interaction effect）
2×2 factorial 設計（兩顧問一致）：
高籌碼 = 0高籌碼 = 1
低基期 = 0A（都不符）B（純高籌碼·高基期追高）
低基期 = 1C（純低基期·可能是死魚）D = LBCA（實驗組）
交互效應要看 (D−C) − (B−A)，或等價的回歸式：
forward_return = α + β1·LowBase + β2·ChipAccel + β3·(LowBase×ChipAccel) + controls + industry FE + date FE
真正要看的是 β3 是否顯著 > 0（而非 D 組平均報酬最高）。controls 至少含 size／流動性／波動／過去20D・60D 報酬／產業／regime／漲停接近度。
Gemini 版指標：Intersection Alpha = Return(D) − Max(Return純低基期, Return純高籌碼)，在 5D/10D 顯著為正且 IR 提升才算過關。
五道強制檢查（codex）：① 5/10/20 日都要有 edge，不能只靠少數飆股撐；② 扣漲停買不到＋隔日開盤滑價後仍有效；③ 空頭 regime 是否直接失效（低基期空頭一樣補跌）；④ 是否集中在少數產業/題材；⑤ 入選後籌碼是否延續、還是事件日後就反轉倒貨。另做 matched portfolio（配對同產業/市值/流動性/年報酬分位但無籌碼加速）對照。
⑥ 建議開工路徑（待 CIO 拍板）
1. 先建特徵、不建池：把 LowBase_Safe 與 ChipAccel_Smoothed 兩組特徵算出來（原料現成：價格庫算年報酬分位/乖離/布林帶寬；CMG 算多視窗 z 階梯）。
2. 2×2 factorial 回測驗 β3：先證明有交互效應，再談建池。沒有顯著 β3 就停手——這一步是存亡閘門。
3. 獨立性四招都要進回測：特別是 LBCA_ex_legacy 版（排既有榜單）與殘差測試，證明不是換皮。
4. 過關才建池 → shadow → 累積 → 有 edge 才升 Tier-2（照 inst_entry_pool 同款分級）。定位＝候選池第 6 入池法。
與現有系統的分工：inst_entry_pool 抓「法人合計已進場」的中大型；LBCA 抓「法人還沒進、在地分點悄悄吸貨」的中小型低基期——理論上是更早、更獨立的一層雷達，互補而非重疊。
⑦ 止跌 gate 定案（顧問三題 + CIO 的 MA20 加碼）
選型定案
時間框週K 20-26 根主判 + 月K 背景濾網（不惡化即可、非硬確認）；日K 不當結構框（低基期股常週K先止跌、月K還醜時才有 alpha）
統計方法Kendall τ 判方向（gate）+ Theil-Sen / log-OLS slope 給幅度（排序、濾死魚股）；不單用 OLS（易被長黑/長紅拉歪）

下面是我會怎麼批判 LBCA v1。先講結論：這個法子有機會，但最大風險是把「弱勢股反彈」包裝成「低基期補漲」，以及把「籌碼訊號的短期抖動」誤判成「大戶加速吸貨」。核心要證明的不是它有沒有正報酬，而是它是否能抓到「價格尚未反映、籌碼已經轉強」的早期段。

1. 低基期口徑推薦

我不建議單獨用「52週位置 < 40%」。這最容易踩價值陷阱，因為長期下跌股天然會落在 range 低段，但那不代表低基期，只代表趨勢爛。

三者排序我會這樣排：
1. 年報酬分位落後 + 非破底止跌
2. 距52週高點乖離
3. 52週range位置

推薦主定義：
  低基期 = 過去一年報酬率在市場後30%
         且 距52週高點乖離 > 20% 或 30%
         且 近60日不再創120日新低
         且 收盤價站回MA60 或 SMI/SHSL confirmed SL

理由：年報酬分位比較能抓「相對沒漲到」，符合補漲邏輯。距52週高點乖離可補強空間感，避免只是盤整低位。52週range位置最危險，因為一路破底的股票也會一直在低段，而且 range 會隨著新低下移，產生假穩定感。我會把「52週位置」降級成輔助條件，不當主條件；比較好的用法是 52週位置介於20%~60%，不要低於20%（太低反而可能是破底股）。你要的是「低基期後轉強」，不是「還在地板下挖洞」。

止跌條件不要只用站上 MA60。MA60 有兩個問題：一是落後，二是容易被反彈假突破騙。建議至少加一個「不再惡化」條件：20日低點 > 前20日低點，或近60日未創120日新低，或 MA20斜率轉正且收盤站上MA60。也就是低基期要分成兩層：低基期空間(過去一年相對弱、距高點夠遠) + 低基期安全(停止破底、短中期結構轉穩)。

2. 加速度訊號怎麼平滑

不要直接拿日頻二階差分。這會製造大量假陽性，尤其台股分點與法人買賣常有單日脈衝、隔日反沖、月底作帳、除權息與題材日噪音。

我會把 chip_momentum 的「加速」改成三層確認：level(平滑後籌碼流為正) / slope(平滑後一階動能轉強) / accel(一階動能本身持續改善)。

具體推薦：
  flow_s = MA5(raw_cmg) 或 EMA5(raw_cmg)
  mom_s  = MA10(flow_s - MA20(flow_s))
  z_mom  = zscore(mom_s, 60)
  acc_s  = MA5(z_mom - lag(z_mom, 5))
觸發條件不要寫成單日：z_mom > 0 且 acc_s > 0 且 z_mom 連續3日改善(或5日內至少4日改善) 且 近10日累積CMG > 近60日分位數的70%。

更穩的版本：CMG_MA20_z60 > 0 且 Δ(CMG_MA20_z60, 5日) > 0 且 Δ(CMG_MA20_z60,10日) > Δ(CMG_MA20_z60,20日)/2。重點是不要追求真正的「二階微分」；你要的是「籌碼改善速度正在變快」，用多視窗斜率比日頻二階可靠。我會避免 today momentum > yesterday momentum、today acceleration > 0 這種訊號（日頻大多是在交易噪音上做微積分）。

更實務的加速定義：短窗籌碼強度 > 中窗 > 長窗，且三者都經過z-score標準化。例如 CMG_5D_z60 > CMG_20D_z60 > CMG_60D_z120 且 CMG_5D_z60 > 1 且 CMG_20D_z60 > 0。這比「二階 > 0」可控很多，也更像你說的 1-2-4 加速。

3. 和現有池子的重疊與冗餘

會高度重疊，尤其和這幾個：主力進場榜(LBCA 的高籌碼增幅很可能只是主力榜的低位版)、inst_entry_pool(若法人合計已轉強，LBCA 很容易變成法人進場加低基期 filter)、SMI 相位(若止跌用 SMI/SHSL confirmed SL，LBCA 可能吃到 SMI 的訊號而不是自己有 alpha)。

所以你要明確規定 LBCA 的獨立定位：它不是找最強籌碼，而是找「價格仍低、籌碼剛轉向、尚未進入既有榜單擁擠區」的股票。我會加三個設計：

第一，做「早期約束」：事件日前20日不得已進入主力進場榜Top N / inst_entry_pool / 投信認養榜。然後觀察事件日後 5/10/20 日是否進入那些榜。如果 LBCA 真的是早期訊號，應該看到：LBCA先出現 → 之後才進主力榜/法人池 → 價格跟上。

第二，用殘差化測試獨立性。把 LBCA 訊號對現有因子做 regression / ranking residual：LBCA_score ~ 主力進場score + inst_entry_score + SMI_phase_score + liquidity + momentum，看 residual 是否還能預測 forward return。如果 residual 沒有 edge，LBCA 就是換皮。

第三，把 LBCA 拆成兩版：LBCA_raw = 低基期∧高籌碼增幅∧止跌；LBCA_ex_legacy = LBCA_raw 但排除近期已入現有榜單者。真正有價值的是 LBCA_ex_legacy。如果只有 LBCA_raw 有效、LBCA_ex_legacy 無效，那它大概率只是現有主力/法人訊號的低位包裝。

4. 回測怎麼證明 interaction effect

不要只比「交叉組 vs 只低基期 vs 只高籌碼 vs 全市場」，這不夠——兩個 filter 串起來本來就會提高品質，未必有交互作用。你需要 2x2 factorial design：
                 高籌碼=0        高籌碼=1
  低基期=0          A               B
  低基期=1          C               D(=LBCA)
interaction 要看 (D - C) - (B - A)，或等價地：D 的超額報酬是否大於 低基期效果 + 高籌碼效果。

回歸式：forward_return = α + β1·LowBase + β2·ChipAccel + β3·LowBase·ChipAccel + controls + industry FE + date FE。你真正要看的是 β3 是否顯著 > 0，而不是 D 組平均報酬是否最高。controls 至少要有 size / liquidity / volatility / past 20D return / past 60D return / industry / market regime / price limit proximity / borrow constraint。

另外要做 matched portfolio，不然 LBCA 可能只是選到小型低流動性股：每檔LBCA股票，配對同產業、相近市值、相近流動性、相近一年報酬分位、但無高籌碼增幅的股票，然後比 forward return / hit rate / drawdown / MFE / MAE。

我會強制加這幾個檢查：(1)D組是否在5/10/20日都有edge，還是只靠少數20日飆股撐起來；(2)扣除漲停買不到、隔日開盤滑價後是否還有效；(3)空頭 regime 是否直接失效；(4)是否集中在少數產業或題材週期；(5)入選後籌碼是否延續、而不是事件日後反轉賣出。最重要的驗證不是平均報酬，是這個：LBCA事件後，既有主力/法人/SMI訊號是否「後續接力確認」。如果 LBCA 後面沒有被更強籌碼訊號接上、只是價格短彈，那它比較像反彈交易，不是早期吸貨模型。

我會給 v1 的修正版：LBCA = LowBase_Safe ∧ ChipAccel_Smoothed ∧ EarlyStage ∧ Tradable
  LowBase_Safe：一年報酬率市場分位<30%；距52週高點>25%；近60日未創120日新低；MA20斜率轉正；收盤站上MA60 或 SMI/SHSL confirmed SL。
  ChipAccel_Smoothed：CMG_MA20_z60>0；Δ(CMG_MA20_z60,5日)>0；CMG_5D_z60>CMG_20D_z60>CMG_60D_z120；5日內至少3日主力買方家數差為正。
  EarlyStage：過去20日未進主力進場榜Top N；未進inst_entry_pool；未進投信認養榜；過去20日漲幅不可已超過市場前15%。
  Tradable：排除處置、全額交割、流動性不足、連續漲停；注意股可單獨分層。

批判結論：LBCA 不該被設計成「低位 + 籌碼強」，這太粗；它應該被設計成「低位止跌後，籌碼由弱轉強，而且尚未被既有強勢榜單捕捉」。否則它不是新選股法，只是主力進場榜加一個低基期濾網。

針對你的新選股法「LBCA v1」，直接切入核心痛點。以下是針對設計缺陷、踩雷點與回測方法的具體批判與修正建議：

1. 低基期用哪個口徑最不踩價值陷阱？

在台股，光靠價格指標(52週位置、乖離)定義低基期，極容易踩到「產業結構性衰退」或「基本面暴雷」的價值陷阱(例如去年的航運、面板、或特定IP股跌深還能更深)。

- 次之：52週價格區間百分位(<40%)。容易在底部盤整極久(時間成本極高)，或遇到下市風險。

為什麼？與防雷修正：(1)市場相對落後段能確保「真的沒漲到」，符合補漲邏輯。(2)必須放棄純 MA60 作為止跌條件——台股常有假突破 MA60 後立刻破底的誘多陷阱。(3)修正提案(加入「籌碼/波動雙重壓縮」)：低基期=一年報酬率<30百分位；止跌過濾器(硬限制)=近20日 Bollinger Band Width 壓縮至近一年新低(代表籌碼沉澱、沒人想賣)，且股價站在週線(MA5)與月線(MA20)之上且兩線黃金交叉。這比 MA60 更敏銳，且能過濾掉死魚股。

2. 加速度(二階)日頻噪音大，該怎麼平滑？

籌碼流向(特別是買賣家數差、主力買超)在日頻的二階導數簡直是隨機雜訊。台股隔日沖主力、假外資分點頻繁製造單日籌碼暴衝，隔天立刻倒貨，會讓你的「1-2-4 加速」滿是假陽性。
具體方案：(1)雙 EMA 濾波：定義 M_t = EMA(CMG_Momentum,5)(先對一階力道做5日平滑)，加速度 A_t = M_t - EMA(M,5)。(2)引入 Z-Score 動態閾值：不要只看 A_t>0(太容易達成)，必須是 A_t 的 Z-Score(過去60天滾動)>1.5，代表顯著的異常加速吸貨而非日常波動。(3)量能背書：加速當日成交量必須 > 5日均量 1.5倍(無量拉抬的籌碼加速多為關聯分點對敲，無效)。

3. 會不會與現有榜單高度重疊？如何抓到更早、更獨立訊號？

極高機率重疊。投信與外資(你的 inst_entry_pool)一旦進場往往連續買進，這會自動觸發你的「籌碼加速」。最後 LBCA 選出來的可能只是 inst_entry_pool 裡的「落後補漲子集」，沒有獨立存在的價值。
如何確保獨立性與早期性：(1)分點特徵去中心化(去機構化)：在 B 條件中排除三大法人買超，專注於「在地分點(地緣券商)」或主力分點六特徵中的特定特徵(如低周轉率大戶、囤貨分點)。如果一檔低基期股，三大法人還沒買，但關鍵少數分點在悄悄加速(買賣家數差極度集中)，這才是真正的早期璞玉。(2)市場規模分流：inst_entry_pool 偏好中大型股(市值>100億)；將 LBCA 限制在市值 20億~80億 的中小型股——這類股票法人受限流動性不能買，但主力分點(內資)能點火，最容易產生「低基期+爆發性補漲」，與現有系統完美互補。

4. 回測對照組怎麼設計，才能證明有 Interaction Effect？

很多多因子回測的騙局在於：因子A很好、因子B一般，疊加後表現好只是因為A在撐，根本沒有 A×B 協同。你必須證明 R(低基期∩高籌碼) > R(低基期)+R(高籌碼)-R(基準)。
四組回測矩陣(Forward Return 5/10/20日)：
- 實驗組(LBCA)：低基期(A)且籌碼加速(B)且止跌(C)。
- 對照組1(純低基期)：A與C但籌碼無加速(甚至流出)。目的：證明籌碼加速是啟動低基期股的催化劑，否則只是抱死魚。
- 對照組2(純籌碼加速)：B與C但股價在高基期(一年報酬>70百分位)。目的：證明低基期提供安全邊際與補漲空間；高基期+籌碼加速易流於短線追高被割。
- 對照組3(隨機/大盤基準)：全市場符合流動性與止跌條件的隨機採樣。
核心檢驗：Intersection Alpha = Return實驗組 - Max(Return對照1, Return對照2)。若在 5D、10D 顯著為正且 Information Ratio 顯著提升，才證明此交叉法有獨立成池的 edge。Regime-conditional：空頭/盤整市中這套的「低基期」防禦屬性是否顯著？(通常這種策略在多頭末段補漲市最強，空頭市防禦力須被檢驗，否則低基期一樣補跌)。
🎯 最終判決　LBCA 未過存亡閘門，不建。低基期是動能逆風（−4.5%）、籌碼加速救不了落後股（D 格 −2.1%～0、交互 β3 負/零）。籌碼的真 edge 在「非落後股」（B 格 +3.5%），已由 inst_entry_pool 捕捉。收斂＝不建 LBCA；止跌 gate 的價值在賣方（結構破壞→出場/否決訊號，非重疊穩健 −3.4%）不在買方（買訊三度撞牆），另開 session 接 discipline 層；文件「低基期補漲」判定為倖存者偏誤。這趟用數據擋下一個看似合理、實則賠錢的策略——信回測不信口號。
