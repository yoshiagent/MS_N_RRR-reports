一、每日自動執行（已設定，不需手動）

Windows Task Scheduler 任務：MS_N_RRR_Daily

  • 時間：週一~週五 18:00（台股收盤 + 30 分鐘）
  • 動作：執行 python -m src.orchestrator
  • 日誌：data/logs/orchestrator_{YYYYMMDD}.log
  • 產出：報告 + GitHub Pages + Telegram + Email

查看狀態

Get-ScheduledTaskInfo -TaskName MS_N_RRR_Daily       # 看上次執行時間
Get-ScheduledTask -TaskName MS_N_RRR_Daily           # 看是否啟用

立即手動觸發排程

Start-ScheduledTask -TaskName MS_N_RRR_Daily

暫停 / 啟用

Disable-ScheduledTask -TaskName MS_N_RRR_Daily       # 暫停
Enable-ScheduledTask  -TaskName MS_N_RRR_Daily       # 恢復
Unregister-ScheduledTask -TaskName MS_N_RRR_Daily    # 完全移除

重新註冊

cd C:\CludeHome\projects\MS_N_RRR
powershell -ExecutionPolicy Bypass -File scripts\install_scheduler.ps1

五、通知系統（Email + Telegram）

配置檔位置

config/notify.local.yaml（不入 git；範本參見 config/notify.example.yaml）

格式：

telegram:
  enabled: true
  bot_token: "<YOUR_BOT_TOKEN>"     # 由 @BotFather 取得
  chat_id:   "<YOUR_CHAT_ID>"       # 由 getUpdates 取得

email:
  enabled: true
  smtp_host: smtp.gmail.com
  smtp_port: 587
  smtp_user: "<your-email@gmail.com>"
  smtp_pass: "<YOUR_APP_PASSWORD>"   # 16 字元 Gmail App Password
  to:        "<your-email@gmail.com>"

⚠ 絕不要把實際 token / password 寫進 OPERATIONS.md、INTEGRATIONS.md 或任何會被部署到 GitHub Pages 的檔案。本檔在 orchestrator 跑時 會被 _sync_docs_to_output() 轉成 HTML 推上公開 repo。 真實憑證只放在 config/notify.local.yaml（已在 .gitignore）。

測試通知

python -c "from src.notify import send_daily; send_daily()"

Telegram 重設 bot token

  1. 開 Telegram 找 @BotFather
  2. /newbot → 取新 token
  3. 開啟新 bot 隨便傳一則訊息給它
  4. 瀏覽器開 https://api.telegram.org/bot<新TOKEN>/getUpdates
  5. 找 "chat":{"id":...} 取 chat_id
  6. 更新 config/notify.local.yaml

Email 重設 Gmail App Password

  1. 開 https://myaccount.google.com/apppasswords
  2. 建立新「應用程式密碼」（16 字元）
  3. 更新 config/notify.local.yaml 的 smtp_pass（去掉空格）

暫停通知

編輯 config/notify.local.yaml：

telegram:
  enabled: false
email:
  enabled: false

七、新增 / 修改 Agent

加新 Agent

  1. 在 src/agents/xxx.py 寫實作（讀中央 stockHisData、寫 data/output/xxx.json）
  2. 在 agents/06_xxx.md（或對應 step 編號）寫規格
  3. 在 src/orchestrator.py 加 step： python from src.agents import xxx as xxx_agent _, err = _run_step(logger, "11. New Agent", xxx_agent.run) summary_meta["steps"]["new_agent"] = "OK" if not err else "FAIL"
  4. 在 src/agents/index_builder.py 加對應卡片 / 連結
  5. 跑 python -m src.orchestrator --skip-fetch --skip-notify 驗證

改既有參數

  • 市場結構閾值：config/parameters.yaml 的 market_exit_threshold / market_warning_threshold
  • RR/RRR Z 門檻：config/parameters.yaml 的 zscore_upper / zscore_lower（但 RR 會被 dynamic_z_threshold 覆蓋）
  • 規則：config/rules.yaml（不做空、單股部位上限等）
  • 股池：config/stock_pool.yaml

九、故障排除

報告沒更新

  1. 查排程器：Get-ScheduledTaskInfo MS_N_RRR_Daily
  2. 看日誌：data/logs/orchestrator_{今天}.log
  3. 手動跑：python -m src.orchestrator --skip-fetch --skip-notify，看哪個 step FAIL

Telegram 未收到

  1. 確認 config/notify.local.yaml 的 telegram.enabled: true
  2. 測試：python -c "from src.notify.telegram import test; test('hi')"
  3. 看是否 bot 被 Telegram 鎖（換新 token，見 § 五）

Email 未收到

  1. 確認 Gmail App Password 還有效
  2. 測試：python -c "from src.notify.email import test; test()"
  3. 看 Gmail 是否進了 spam

某配對沒產 chart

  1. 看 data/output/custom_pairs.json 的 chart_status：
  2. built → 成功
  3. from_xlsx → 用 xlsx URL（沒有跑內建 build）
  4. build_failed: xxx → 看 error 通常是缺資料
  5. 若是缺資料：python -c "from src.shared import data_loader as dl; dl.load_price('XXXX')"
  6. 若需抓新個股：python -c "from stock_data import fetch_price; fetch_price('XXXX')"

主頁進階指標掉資料

  • 看 market.json 是否有 regime 欄位
  • 沒有 → structure_zscore 模組失敗，跑 python -m src.shared.structure_zscore debug
  • 通常是中央 processed/3insti/market_aggregate.csv 或 processed/options/foreign_opt.csv 過舊

orchestrator 卡在某 step

  • 看 data/logs/orchestrator_{date}.log 最後幾行
  • 多半是網路 timeout（yfinance / TWSE）
  • 解法：--skip-fetch 跑一次，確認後續 step 正常

git push 失敗

  • cd data/output && git status 看是否有未 commit
  • 若 remote rejected：git pull --rebase origin main 後再 push
