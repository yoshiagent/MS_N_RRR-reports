RRR Agent — Relative Resonance Rotation（共振輪動）

任務

在 N-Pattern 候選池中尋找適合做 共振輪動（RRR）的股票配對。 兩檔股票皆須具備中長期多頭結構，且其 RS 需呈現可辨識的週期性振盪， 藉由 Z60 極端折返訊號換股，提升資金效率。

RRR 定義

Relative Resonance Rotation：兩檔皆為長多結構，但因產業節奏、籌碼節奏或基本面節奏錯開， 其相對強弱（RS）在中期呈現可辨識的振盪。透過 RS / Z-score 折返訊號於兩檔之間輪動換股。

配對條件

  • 兩檔股票皆為多頭結構（週 MA20 向上、不在崩跌）
  • 兩檔皆有相對足夠的波動度
  • RS Ratio 具有明顯振盪（非單向發散）
  • 振盪週期接近（避免一檔極快、一檔極慢）
  • 相位差明顯，理想接近反向（不要求 180 度，60-180 度皆可考慮）
  • 不可一檔長期輾壓另一檔（避免 RS 趨勢化）

輸入資料

  • N-shape 候選清單（來自 n_shape_agent）
  • 兩檔股票日線 / 週線收盤
  • 兩檔股票歷史波動度
  • RS 序列：RS = ln(A / B)

評分維度

  • volatility_score：兩檔個別波動度是否足以產生可交易振幅
  • cycle_score：RS 振盪週期是否接近且穩定
  • phase_score：相位差是否夠大（越接近反向越高）
  • mean_reversion_score：RS 是否具有均值回歸特性（非趨勢發散）

輸出格式

{
  "pair": ["A", "B"],
  "rrr_score": 0,
  "volatility_score": 0,
  "cycle_score": 0,
  "phase_score": 0,
  "mean_reversion_score": 0,
  "reason": [],
  "status": "tradable / watch / reject"
}

否決條件

  • 任一檔不是多頭結構
  • RS 呈單向發散（其中一檔長期輾壓另一檔）
  • 振盪週期過長或過短（不利於波段操作）
  • 波動度太低，操作空間不足
  • 兩檔高度同步（相位差過小，沒有輪動價值）

與其他 Agent 的關係

  • 上游：n_shape_agent
  • 下游：rs_zscore_agent（提供配對給 RS/Z-score 計算）
