HotFlow Agent（主流形成偵測 Agent）

任務

每日從台股全市場（~1,935 檔）找出「可能正在形成主流資金流」的候選股， 作為後續 N型結構 Agent、RRR 輪動 Agent、風控 Agent 的候選池來源。

本 Agent 不做買賣決策、不直接產生進出場訊號。

在 MS_N_RR/RRR Ver.A 中的位置

Market Structure        ← 總開關（市場狀態）
        ↓
HotFlow Agent           ← 候選池產生（本 Agent，1,935 → ~30-50）
        ↓
N-shape Agent           ← 結構確認（候選池 → 多頭 N 型）
        ↓
RRR Pair Agent          ← 配對篩選
        ↓
RS / Z-score Agent      ← 換股訊號
        ↓
Chip Filter             ← 籌碼濾網
        ↓
Risk Control            ← 守門員
        ↓
Execution Report        ← 最終輸出

一、資料輸入

由 Data Agent 統一供給（C:\CludeHome\stockHisData）：

程式欄位 中文 來源 / 計算
date 資料日 price.date
stock_id 代號 price.symbol
stock_name 股名 JOIN symbols.name
close 收盤 price.close（已 auto-adjust）
volume 成交量 price.volume（股數）
amount 成交金額 = close × volume
amount_rank 成交金額名次 ROW_NUMBER() OVER (PARTITION BY date ORDER BY amount DESC)
market_index_close 大盤收盤 JOIN indices(name='TWII').close
foreign_buy_shares 外資淨股數 institutional."foreign"
investment_trust_buy_shares 投信淨股數 institutional.trust
dealer_buy_shares 自營淨股數 institutional.dealer

我們以淨股數而非成交金額計算法人指標（資料層只存股數）。

二、核心衍生欄位

程式欄位 中文 計算公式
rank_change 名次變動 previous_amount_rank − current_amount_rank（正值 = 名次上升）
rank_score 名次分數 第 1 名 = 100，第 100 名 = 1，未進榜 = 0
rank_momentum 名次動能 rank_score_today − rank_score_yesterday
rank_slope_5d 5日名次斜率 最近 5 日 rank_score 的 OLS 斜率（向量化：(−2y₋₄ −1y₋₃ +1y₋₁ +2y₀)/10）
hot_persistence_20d 20日上榜天數 近 20 日進入 amount_rank ≤ 100 的天數
amount_expansion 成交金額擴張倍數 今日 amount ÷ 過去 60 日平均 amount
amount_z_60d 成交金額 60 日 Z 分數 (today − ma60) / std60
rs 相對強弱 RS 個股收盤 ÷ 大盤收盤
rs_ma20 RS 的 20 日均 rs 的 20 日 rolling mean
rs_trend RS 趨勢 rs_ma20 今日 > 昨日 → 1，否則 0
weekly_ma20_slope 週MA20 斜率 日 K → 週 K resample → 週MA20 → 最近 5 週 OLS 斜率

三、HotFlowScore 計算（加權合成）

子分數 權重
RankMomentum 25%
RankSlope_5D 20%
HotPersistence_20D 15%
AmountZ_60D 15%
AmountExpansion 10%
RS（RS_Trend × 50% + RS 強度 × 50%） 10%
WeeklyMA20Slope 5%

正規化流程： 1. Winsorize：對每個原始指標 clip 在 [P1, P99]，避免極端值 2. Percentile rank：用 rank(pct=True) × 100，得到 0–100 的截面排名 3. 加權合成：HotFlowScore = Σ (pr_i × weight_i)（範圍 0–100）

全部用百分位排名（cross-section within today's universe），避免不同指標尺度差異。

四、硬性前置條件（任一未過 → 直接排除）

條件 理由
歷史資料 ≥ 100 日 算 60 日 Z 分數 + 24 週 MA20
今日 volume > 0 排除停牌
20 日均成交金額 ≥ 5,000 萬 NTD 流動性下限，避免小型股雜訊
週 MA20 斜率 > 0（最近 5 週 OLS） 中期動能向上是基本盤

五、分類規則

A 主流形成中

全部條件均成立： - HotFlowScore 位於前 20%（percentile rank ≥ 80） - rank_slope_5d > 0（名次趨勢上升中） - hot_persistence_20d ≥ 3（20 日內至少 3 次上榜） - rs_trend == 1（RS 正在轉強） - weekly_ma20_slope ≥ 0（週MA20 不下彎） - 近 5 日漲幅 ≤ 25%（未過熱）

B 觀察中

  • HotFlowScore 位於前 40%（PR ≥ 60）
  • 但未完全符合 A 級條件

C 過熱或排除

任一觸發即歸 C： - 近 5 日漲幅 > 25% - amount_z_60d > 3 但 hot_persistence_20d < 2（單日爆衝缺乏持續度） - rs_trend == 0 且 rs_strength_20d < 0（RS 未轉強） - weekly_ma20_slope < 0（週MA20 下彎）

八、與其他 Agent 的關係

  • 上游：market_structure_agent
  • 市場狀態為 exit 時，HotFlow 仍會跑但下游 risk_agent 不會放行
  • 市場 defensive 時，可考慮提高 A 級 PR 門檻（從 80 → 90）
  • 下游：n_shape_agent
  • n_shape 改用 HotFlow 的 A/B 候選清單作為輸入（取代 stock_pool.yaml 的固定 61 檔）
  • 動態股池：今日 A + B 才進 N-shape 評估

九、未來擴充方向

  1. 產業集中度偵測：A 級 candidate 在同一產業多檔同時出現 → 標「主流族群」
  2. 主題自動標籤：AI 伺服器 / ABF / CCL / 散熱 / 記憶體… 等
  3. 資金流時序：HotFlowScore 的 5 日趨勢（上升 / 平穩 / 下降）
  4. 悄悄放量 vs 天量爆發：低 amount_z_60d 但高 hot_persistence_20d → 標「早期主流」
  5. 主力分點淨買賣：當作 chip 子分數，需資料源接入
  6. 借券大增警示：硬扣分（潛在放空壓力）
  7. 回測：用歷史 HotFlowScore 預測未來 N 日報酬，校準權重與門檻
