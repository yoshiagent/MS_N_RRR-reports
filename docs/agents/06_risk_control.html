Risk Agent（風控守門員）

任務

最後決策守門員。決定是否允許換股、允許多少倉位、允許多少批次。 所有換股訊號最終必須通過 risk_agent 才會出現在 execution_agent 報告中。

核心規則

  • 不做空（allow_short = false，硬性規則）
  • 只在配對中兩檔皆「週 MA20 向上」時，才允許 RRR 換股
  • 市場結構為 defensive 時，降低換股頻率（提高訊號門檻 / 縮小單次部位）
  • 市場結構為 exit 時，不產生積極換股建議
  • 單一股票部位不得過度集中（依 rules.yaml 設定上限）
  • 支援分批換股（25% / 50% / 75% / 100%）
  • 價差 Z-score 未折返，不追訊號

輸入資料

  • market_structure_agent 輸出
  • rs_zscore_agent 輸出
  • chip_agent 輸出
  • 目前持倉與部位比例
  • 參數：position_steps、單股上限、配對上限

輸出格式

{
  "allowed": true,
  "position_action": "full_switch / partial_switch / hold / reduce / stop",
  "position_size": "0% / 25% / 50% / 75% / 100%",
  "risk_reason": []
}

否決條件

  • 市場狀態 = exit
  • 配對任一檔週 MA20 向下
  • 籌碼 Agent 對任一檔給出 reject
  • Z-score 處於極端值但尚未折返
  • 單股已達持倉上限

與其他 Agent 的關係

  • 是所有訊號的最後關卡
  • 直接決定 execution_agent 的「最終行動建議」

未來擴充方向

  • 加入個股波動度權重（高波動股自動降低倉位）
  • 加入帳戶層級的整體曝險上限（總多頭曝險 ≤ X%）
  • 引入回撤控制（連續虧損後自動降批次）
  • 加入「冷卻期」機制（剛換股的配對短期內不重複換）
