Box Structure Agent 規格（股票箱結構分析）

對應實作： - src/agents/box_structure.py — 核心演算法 - src/agents/box_structure_report.py — HTML 渲染

規格類別：結構穩定性 Agent（不預測，只判斷結構品質）

任務

依 K 線結構判斷： 1. 是否存在「有效股票箱」 2. 股票箱是否可作為 N 形延續/觀察/破壞的判斷基準 3. 多個股票箱是否顯示趨勢正在劣化或即將結束

主時間框架：週線（判斷主結構）；日線可作為進出場輔助（未來擴充）

設計偏好： ✓ 裸 K + 成交量 + ATR(20) + SH/SL 結構 ✗ 不依賴 MACD / KD / 過多技術指標

候選池

HotFlow ∪ N-Pattern ∪ 配對表.xlsx ∪ Stock Ops

通常 ~67 檔。

演算法（9 步驟）

Step 1 — 計算 ATR(20)

True Range 的 20 期均值，用於後續閾值。

Step 2 — 偵測候選 pivot

high[i] = max(high[i-2:i+3]) → SH 候選 low[i] = min(low[i-2:i+3]) → SL 候選

Step 3 — 套用「有效性」條件

有效 SH 須同時符合：

條件 規則
A. 前方有效上漲 漲幅 ≥ ATR × 2 或 ≥ 8%
B. 後方有效回落 自高點回落 ≥ ATR × 0.5 或 ≥ 3%
C. 確認期 ≥ 3 根 K 棒未再創高

有效 SL 須同時符合：

條件 規則
A. 前方有效下跌 跌幅 ≥ ATR × 0.5 或 ≥ 3%
B. 後方有效反彈 反彈 ≥ ATR × 0.5 或 ≥ 3%
C. 確認期 ≥ 3 根 K 棒未再破低

Step 4 — Compression 識別

實作上以「箱體高度過小」隱含表達：箱高 < max(ATR/Close × 1.5, 5%) → 分類為 compression，視為原推進延續，不切新 N 形。

Step 5 — 建構有效箱（SH-SL 配對）

對每個 SL，找「自上一個 SL 之後最高的有效 SH」配對。 箱頂必須 > 箱底（合理性檢查）。

每個箱含： - sh_date / sh_price — 箱頂 - sl_date / sl_price — 箱底 - height_pct = (SH − SL) / SL - duration_weeks — SH 到 SL 的週數 - volume_change — 縮量 / 放量 / 持平（vs SH 前 10 週均量） - fake_breakout — SL 後 5 根內突破 SH 又跌回箱內 - breakout_confirmed — 突破 SH 且 5 根內未跌回

Step 6 — 5 類箱體分類

類型 條件 UI 顏色
🟢 健康回檔箱 已突破確認 + 箱底高於前箱 + 箱高適中 綠
🔵 壓縮整理箱 箱高 < 門檻（雜訊區） 藍
🟡 觀察箱 尚未突破 SH 也未跌破 SL 黃
🟠 劣化箱 箱底未上移 / 箱高擴大 / 假突破 橘
🔴 破壞箱 收盤跌破 SL 紅

Step 7 — 多箱劣化分析（比較最近 3 箱）

逐項加分，總分 0-100：

訊號 加分
箱底未持續上移（HL 弱化） +12
最新箱底跌破前箱底（trend down） +15
箱高擴大 > 20% +10
回檔幅度加深超過 5 個百分點 +8
整理時間變長 > 50% +8
每個假突破 +12
箱體重疊度高 +10
當前收盤跌破最新 SL +25
近 3 箱中有 2 個劣化箱 +10
近 3 箱中有 1 個破壞箱 +15

Step 8 — 4 趨勢階段

Stage 條件
🌱 Stage 1 初升段 僅 1 個有效箱
🚀 Stage 2 主升段 箱底上移 + 突破有效 + 劣化 ≤ 50
⚠ Stage 3 高位震盪/劣化 劣化評分 ≥ 51
📉 Stage 4 趨勢破壞 跌破最新 SL，或最新箱底跌破前箱底

Step 9 — 操作建議 + 一句話結論

Stage 持股 候選股
Stage 1 續抱 等回測 SL 不破
Stage 2 / N 延續 續抱 可加入觀察名單
Stage 2 / 壓縮 續抱 等突破壓縮上緣
Stage 3 (劣化 < 70) 減碼 暫不介入
Stage 3 (劣化 ≥ 70) 停利 暫不介入
Stage 4 停損 排除

不會做的事

  • ❌ 不預測股價方向
  • ❌ 不單獨建議買賣（僅評估結構穩定性）
  • ❌ 不依賴 KD / MACD
  • ❌ 不將過小波動視為新 N 形
