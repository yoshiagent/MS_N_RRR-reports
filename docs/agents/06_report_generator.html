Execution Agent

任務

彙整所有上游 Agent 的結果，產生每日或每週摘要報告。 這是使用者實際看到的最終輸出。

報告結構（固定 8 段）

  1. 市場狀態（來自 market_structure_agent）
  2. 可觀察族群（依產業 / 主題分類的強勢族群）
  3. N 型候選股（來自 n_shape_agent，依分數排序）
  4. RRR 配對清單（來自 rrr_pair_agent，依 rrr_score 排序）
  5. 今日 RS / Z-score 訊號（來自 rs_zscore_agent）
  6. 籌碼警示（來自 chip_agent，列出 warning / reject 名單）
  7. 風控建議（來自 risk_agent）
  8. 最終行動建議（彙整後給出可執行清單）

輸入資料

  • 上游所有 Agent 的輸出 JSON
  • 報告週期設定（daily / weekly）
  • 輸出目錄：data/output/

輸出格式

  • 主要：HTML（依使用者規則，文件輸出一律使用 HTML）
  • 附帶：JSON（供其他系統串接）
  • 命名：MS_NRRR_{YYYYMMDD}_{daily|weekly}.html

內容要點

  • 報告必須能讓使用者一頁看完當日決策
  • 「最終行動建議」段必須明確：哪一檔換到哪一檔、換多少比例、依據是什麼
  • 若市場狀態為 exit，最終行動建議須明確顯示「停止積極換股」

否決條件

  • 上游任一 Agent 缺資料時，該段顯示 N/A 並標示原因，不可猜測
  • risk_agent 未放行的訊號，不可出現在「最終行動建議」段

與其他 Agent 的關係

  • 下游無，是最終輸出
  • 不做任何判斷，只做彙整與呈現

未來擴充方向

  • 加入績效歸因（哪個 Agent 的訊號最有效）
  • 加入互動式 HTML（可展開細節、可篩選）
  • 加入 Email / Line / Telegram 推播
