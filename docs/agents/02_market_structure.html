Market Structure Agent

任務

判斷大盤目前處於哪一種結構，作為整個 MS_N_RR/RRR Ver.A 系統的「總開關」。 所有換股建議都必須先經過此 Agent 的允許才能啟動。

市場狀態分類

  • bullish（多頭區）：可積極換股
  • warning（警告區）：可換股，但提高門檻
  • defensive（防守區）：降低換股頻率
  • exit（出清區）：停止所有積極換股

輸入資料

  • 加權指數（TWII）日線、週線、月線
  • 櫃買指數（OTCI）日線、週線、月線
  • 大盤成交量
  • 三大法人（外資 / 投信 / 自營）買賣超
  • 台幣兌美元匯率
  • VIX 恐慌指數
  • 台指期 外資 / 投信 / 自營 未平倉口數（位置 + 5d/20d 趨勢）

判斷依據（第一階段先列原則，第二階段再公式化）

  • 月線、週線結構是否仍維持多頭排列
  • 週 MA20 是否向上
  • 量價配合度
  • 法人是否同步轉空
  • 匯率是否大幅貶值
  • 台指期外資未平倉是否轉為大幅淨空

輸出格式

{
  "market_state": "bullish / warning / defensive / exit",
  "score": 0,
  "reason": [],
  "allowed_action": "aggressive / normal / reduce / stop"
}

否決條件（直接判定為 exit）

  • 大盤跌破週 MA20 且月線轉折向下
  • 法人連續多日大幅賣超且量價背離
  • 出現系統性風險訊號（匯率急貶、期貨外資大量轉空）

與其他 Agent 的關係

  • 是所有 Agent 的上游
  • risk_agent 必須讀取本 Agent 結果決定是否放行換股
  • execution_agent 第一行即為市場狀態

Phase 1：60 日 Z-score 相對歷史模型（內建，2026-05-25 起）

本 Agent 除絕對水位 + 斜率分析外，另內建 60 日滾動 Z-score 模型 （源自姊妹專案 市場結構NRR 的 indicators/market_structure.py）， 產生以下「相對歷史」維度欄位寫入 market.json：

欄位 意義
foreign_pressure_score 外資壓力分數（↑=壓力大），輸入：外資現貨/期貨 Z、USDTWD Z、SOX Z
etf_support_score ETF 承接分數（↑=內資承接強），輸入：投信 Z、ETF 成交占比 Z、跌幅收斂 Z
market_structure_score etf_support − foreign_pressure
regime strong_bull/mild_bull/neutral/mild_bear/strong_bear
dynamic_z_threshold RR 動態 Z 門檻（1.5/2.0/2.5），由 regime + 站上週MA20 決定
taiex_above_wma20 TAIEX 是否站上週 MA20（用 100 日 MA 近似）
raw_values 原始值 dict（外資現貨億元、PCR、USDTWD 等）
z_subscores 各輸入的 60 日 Z 值 dict

實作位於 src/shared/structure_zscore.py。 資料輸入（Phase 4 起完全自主）：全部由中央 stockHisData 提供， 包含 SOX/USDTWD/TWII/外資台指期/大盤三大法人/選擇權 PCR/ETF volume。

下游消費： - rs_zscore.py 讀 dynamic_z_threshold 取代靜態 ±2.0 - index_builder.py 進階指標區塊優先讀 market.json 本身欄位， 姊妹專案 CSV 退為 fallback
