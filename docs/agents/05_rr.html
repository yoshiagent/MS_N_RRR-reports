RR Agent — Relative Rotation（相對輪動）

任務

簡版相對輪動策略。對任意兩檔多頭股票，用 RS 對 RS_MA20 的穿越 觸發換股訊號， 追隨當前較強的一邊。訊號較頻繁、適合趨勢追隨。

與 RRR（共振輪動，見 05_rrr.md）並存： - RR：寬鬆，跟著趨勢走 - RRR：嚴格，要求 RS 有 cycle / phase / 雙邊極端，等折返才換

核心公式

RS         = ln(close_A / close_B)
RS_MA20    = 20 日 rolling mean of RS
RS_MA60    = 60 日 rolling mean
side       = "A" 若 RS > RS_MA20，否則 "B"

訊號邏輯

條件 訊號 含義
RS 在近 5 日從 ≤ MA20 上穿至 > MA20 switch_to_A A 重新轉強，換到 A
RS 在近 5 日從 ≥ MA20 下穿至 < MA20 switch_to_B B 重新轉強，換到 B
RS > MA20 且無近期穿越 hold_A A 持續優勢
RS < MA20 且無近期穿越 hold_B B 持續優勢
資料 / 波動不足 no_pair 不評估

近 5 日穿越視窗（CROSS_WINDOW = 5）可調。

Confidence 計分 (0-100)

base   = min(60, |RS - MA20| / |MA20| × 100% × 20 分)   # 距離強度
bonus  = min(40, |MA20 5日變化| × 80 分)                # MA20 斜率強度
conf   = base + bonus

距離 1%（MA20 與 RS 差 1%）= +20 分，最多 +60；MA20 斜率快 = 加 bonus。

配對篩選條件

  • 共同歷史 ≥ 80 個交易日
  • 兩檔年化波動皆 ≥ 10%

比 RRR 寬鬆很多，幾乎任何流動性 OK 的多頭配對都能評估。

來源 內容
上游：N-Pattern Agent candidate 清單（多頭結構過關的個股）
Data Agent 個股日線 OHLCV（load_prices_bulk 一次撈）

與其他 Agent 的關係

Market Structure
    ↓
HotFlow Agent          ── 篩選主流候選池
    ↓
N-Pattern Agent        ── 通過結構評分
    ↓
┌─────────┐ ┌─────────┐
│ RR      │ │ RRR     │ ← 兩策略並列；用同一候選池但邏輯不同
└─────────┘ └─────────┘
    ↓           ↓
Risk Control（共用）
    ↓
Final Report

與 RRR 對照表

面向 RR（本 Agent） RRR
訊號來源 RS 對 MA20 穿越 Z60 極端折返
配對篩選 寬鬆（波動、歷史） 嚴格（cycle / phase / 雙邊極端）
訊號頻率 較多（趨勢追隨） 較少（等極端反轉）
思維 Trend following Mean reversion
風控 取部位較保守（無 chip 加強） 風控 gate 嚴格
