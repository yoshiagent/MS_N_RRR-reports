Stock Ops Agent 規格（股票操作 Agent）

對應實作： - src/agents/stock_ops.py — 評估邏輯 - src/agents/stock_ops_report.py — HTML 渲染

規格類別：決策層 Agent（整合多個既有 agent，輸出具體操作建議）

任務

在「市場結構偏多」的背景下，找出： 1. 月線剛脫離底部盤整 2. 週線進入主升段 3. 日線健康修正後重新轉強 4. 適合「佈局」與「加碼」的股票

設計偏好（與規格一致）

✓ 波段交易　✓ 裸 K + 成交量　✓ 趨勢結構　✓ Relative Rotation　✓ 不做空　✓ 主流資金流向　✓ 量化 + 結構分析

✗ 避免依賴：KD / MACD / 過多技術指標

候選池

HotFlow A + B 級
  ∪  N-Pattern candidate + watch
  ∪  配對表.xlsx 自選股

通常 30-60 檔，可深度月/週/日三層分析。

10 個檢查模組

Module 1 — 市場結構 Gate（必要條件）

讀取 market.json 的 subscores：

條件 取值 通過閾值
市場狀態 market_state bullish 或 warning
加權週線 twii_weekly ≥ 60
櫃買週線 otci_weekly ≥ 60
外資台指期 foreign_oi ≥ 40

任一不通過 → 所有股票歸為「觀察」階段（部位 0%）。

Module 2 — 月線主升段起點（0-100 分）

對 daily price resample 為 月 K，計算：

項目 權重 邏輯
月 MA10 上彎 35 近 3 個月 MA10 - 5 個月前 > 0
月 K 突破 12 月區間頂部 35 last_close > prev_12m_high
月成交量擴張 20 近 3 月均量 > 過去 12 月均量
月線底底高 10 最新 swing low > 前一個 swing low

≥ 60 分為通過。

Module 3 — 週線主升段確認（0-100 分）

對 daily price resample 為 週 K（W-FRI），計算：

項目 權重 邏輯
週 MA20 向上 30 OLS 對近 5 週 MA20 斜率 > 0
股價 > 週 MA20 25 last_close > last_ma20
Higher High 20 近 12 週最新 swing high > 前一個
Higher Low 15 近 12 週最新 swing low > 前一個
MA20 斜率加速 10 近 5 週 slope > 前 5 週 slope

≥ 60 分為通過。

Module 4 — 日線 Z60 過熱史

bias    = (close - MA20) / MA20
bias_z  = (bias - mean60) / std60

近 60 日 Z60 是否曾達 ≥ +1.5？這代表股票曾有主升段動能。

Module 5 — 日線修正完成

理想 Z60 區間：[0, +0.5] 不建議：Z60 < -1（已轉弱）

Module 6 — 量縮確認

vol_ratio_5d_vs_20d = mean(volume[-5:]) / mean(volume[-20:])
volume_shrink = vol_ratio < 1.0

代表主力未明顯撤退。

Module 7 — 重新轉強訊號（核心）

至少符合其中之一即視為「重新轉強」：

  1. 突破前日高點：today.close > yesterday.high
  2. 紅 K 吞噬：today.close > today.open AND today.high > yesterday.high AND today.low < yesterday.low
  3. 帶量紅 K：紅 K + today.volume > 5d_mean × 1.2
  4. 5 日平台突破：today.close > 過去 5 日最高（不含今日）

外加：Z60 重新上彎（z_today > z_yesterday）— 修正可能結束的關鍵訊號。

Module 8 — 主流性評估

從 hotflow.json 取： - hotflow_score（主流評分） - amount_rank（成交額名次） - lifecycle_stage（4 階段狀態機）

主流性高 = 成交額名次靠前 + lifecycle 處於 forming / expanding。

Module 9 — RR 配對對手

掃描 4 個來源是否有對手： - rr.json (RR 簡版) - rrr.json (RRR 嚴格版) - custom_pairs.json (自選配對) - dynamic_rr.json (動態 RR)

顯示前 5 組相關配對 + 換股方向。

Module 10 — 結構停損（長線保護短線原則）

依優先序： 1. 轉強 K 低點 −1%（若今日為轉強 K） 2. 5 日平台低點 −1%（要求距現價 ≥ 5%） 3. 日 MA20 −1%

選符合條件最近最高者。

5 階段分類邏輯

if not market_gate_ok:
    → 觀察 (0%)

elif monthly ≥ 60 and weekly ≥ 60:
    if daily_state == "修正完成+重新轉強":
        if higher_high + amount_expand + amount_rank ≤ 50:
            → 加碼 (30%)         # 第一次加碼
        else:
            → 佈局 (30%)         # 中間偏進取
    elif daily_state == "主升段持續":
        → 持有 (100%)            # 主升中，不加倉
    elif daily_state == "修正完成中":
        → 佈局 (20%)             # 保守佈局
    elif daily_state == "已轉弱":
        → 減碼 (0%)              # 結構轉弱出場
    elif daily_state == "過熱中":
        → 持有 (100%)            # 不追高

else:
    → 觀察 (0%)

A/B/C 風險等級

A 級（全部條件齊備）：
  ✓ market_gate_ok
  ✓ monthly ≥ 60 AND weekly ≥ 60
  ✓ daily_state == "修正完成+重新轉強"
  ✓ amount_rank ≤ 100（主流性）
  ✓ rr_pair_count > 0（有對手）
  ✓ 失敗風險 ≤ 1 項

B 級（主要達標但有警示）：
  ✓ market_gate_ok
  ✓ monthly ≥ 60 AND weekly ≥ 60
  ✓ 失敗風險 ≤ 3 項

C 級：以上皆未達標

加入 RR 配對觀察池建議

當以下條件成立時，系統建議將該檔加入 配對表.xlsx：

monthly ≥ 60
AND weekly ≥ 60
AND amount_rank ≤ 60（主流性高）
AND rr_pair_count == 0（尚無對手）
AND lifecycle_stage ∈ {forming, expanding}

不會做的事（明確界線）

  • 不執行交易：只給建議，沒有 broker 整合
  • 不做空：嚴格只給多方訊號，「減碼」是退場不是放空
  • 不做下跌攤平：禁止；只允許「突破加碼」
  • 不依賴 KD/MACD：所有判斷都基於價格、量、結構
  • 不取代 RR/RRR Agent：本 Agent 是統合層，仍需 RR/RRR 提供換股訊號
