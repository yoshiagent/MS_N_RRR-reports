RS / Z-Score Agent

任務

計算 RRR 配對的相對強弱（RS）與其 Z-score，產生換股訊號。 這是 MS_N_RR/RRR Ver.A 系統中最核心的換股觸發器。

核心計算

RS = ln(A / B)
rolling_mean = mean(RS, window=60)
rolling_std  = std(RS,  window=60)
Z-score = (RS - rolling_mean) / rolling_std

預設 window = 60（交易日，約三個月）。

訊號邏輯

  • Z-score > +zscore_upper（預設 +2）：A 相對 B 過強，等待折返
  • Z-score < zscore_lower（預設 -2）：A 相對 B 過弱，等待折返
  • 必須等極端值「折返」進入緩衝區（zscore_reentry_buffer，預設 0.3）內， 才確認換股訊號
  • 不在極端值未折返時追訊號
  • 支援分批換股（依 position_steps 分批，例如 25% / 50% / 75% / 100%）

輸入資料

  • RRR Pair Agent 通過的配對清單
  • 兩檔股票對齊後的日線收盤序列
  • 參數：rs_window、zscore_upper、zscore_lower、zscore_reentry_buffer

輸出格式

{
  "pair": ["A", "B"],
  "rs_value": 0.0,
  "zscore": 0.0,
  "signal": "hold_A / hold_B / switch_to_A / switch_to_B / no_action",
  "confidence": 0,
  "reason": []
}

否決條件（不發出 switch 訊號）

  • Z-score 仍在極端值且未折返
  • 配對中任一檔已被 n_shape_agent 降級為 reject
  • 市場狀態為 exit
  • 籌碼 Agent 給出 reject

與其他 Agent 的關係

  • 上游：rrr_pair_agent（提供可交易配對）
  • 下游：chip_agent、risk_agent（最終是否執行由 risk_agent 決定）

未來擴充方向

  • 多 window 投票（30 / 60 / 120）提高訊號穩健度
  • 動態門檻：依配對波動度自動調整 zscore_upper / lower
  • 加入 RS 動量斜率（折返速度）作為 confidence 加權
  • 引入分批換股建議比例輸出（不只 signal，還給出建議倉位變化）
