MS-Agent v2 規格（Market Structure Agent v2）

對應實作： - src/agents/ms_breadth.py - src/agents/ms_diffusion.py - src/agents/hotflow_lifecycle.py - src/agents/ms_temperature.py - src/agents/ms_risk.py - src/agents/ms_dashboard.py

規格類別：整合決策層 Agent（合成多個既有 agent 的輸出）

任務

「市場結構監控中心」 — 不預測明天漲跌，持續監控資金如何流動， 提前發現主流、提前發現主流退潮、找出 RR 最佳環境、找出末升段風險。

設計理念：

偏向「資金流分析」而非單純技術分析。 不依賴 KD / MACD / 單一技術指標。 優先觀察：資金流、成交金額、市場結構、主流擴散、Breadth、法人方向。

M2 — 市場廣度（ms_breadth.py）

4 指標 → 合成 breadth_score (0-100)：

指標 計算 權重
advance_decline_ratio 今日上漲家數 / (上漲 + 下跌) 0.30
new_high_20d_count 收盤 ≥ 過去 20 日 max 的家數比 0.25
above_ma20_pct 收盤 > MA20 的全市場比例 0.30
otc_strength OTC 站MA20 比 - TWSE 站MA20 比 0.15

4 級分類：healthy ≥ 70 / mixed ≥ 50 / weak ≥ 30 / panic < 30

資料源：DuckDB price 表（全市場 ~1,935 檔，最近 60 個交易日）

選 as_of 邏輯：取最近一個「全市場齊全（≥ 1000 檔）」的交易日，避免取到只有 universe 子集的最新日。

M3 — 資金擴散（ms_diffusion.py，核心權重最高）

5 指標 → 合成 diffusion_score (0-100)：

指標 計算
new_in_top100_5d 今日 Top100 ∩ 5 日前 Top100 的差集大小
smallcap_pct Top100 中市值 < 1000 億的比例（XQ 真實市值）
sector_hhi XQ 細產業 545 種的 HHI 指數（主要 HHI）
theme_hhi 自建主題 HHI（次要參考）
ai_sector_pct AI 族群佔 Top100 成交額比例

4 級分類（依 diffusion_score + HHI 綜合判斷）： - over_concentrated — HHI ≥ 2500（不管 score 多少） - expanding — score ≥ 65 - stable — score ≥ 40 - narrowing — score < 40

HHI 設計重點：

用 XQ 細產業（545 種） 算 HHI 而非官方 28 大類。 因半導體 + 電子零組件 天然佔台股大半，用官方算永遠 over_concentrated，失去敏感度。 XQ 細產業範圍：< 500 真正分散、500-1500 健康、> 2500 才警示。

資料源： - DuckDB price 表（最近 10 日 Top 300） - stockHisData/database/sector_map.json（三軌產業 + XQ 市值）

M4 — HotFlow Lifecycle（hotflow_lifecycle.py）

升級舊版 A/B/C 靜態分類為「4 階段生命週期」狀態機。

5 狀態（依優先序判斷）：

階段 條件
overheating 🔥 score_pr ≥ 95 且 5 日漲幅 ≥ 20%
fading 📉 5 日 hotflow_score 斜率 ≤ -5 且 20日上榜下降 ≥ 2
expanding 📈 20 日上榜 ≥ 7 日 且 同 XQ 細產業 ≥ 3 檔同步入榜
forming 🌱 5 日 hotflow_score 斜率 ≥ +5 且 20 日上榜 ≤ 9
dormant 💤 以上皆不符

狀態持久化： - 跨日 state：data/processed/hotflow_lifecycle_state.json - 每檔記錄 stage / stage_first_seen / score_history[5d] / persist_history[5d] - 自動清理 30 日內未見的 stale 檔 - 每日歷史：data/processed/lifecycle_history.csv - 每行：date / forming / expanding / overheating / fading / dormant - 給未來「主流形成 → 退潮全過程」回測使用

整合進 hotflow.html：每張卡片右上加 lc-pill 顯示階段（含「第 N 日」）。

M6 — 大盤層級風險（ms_risk.py）

7 訊號（每觸發一個 +1）：

編號 訊號 觸發條件
1 高檔爆量 TAIEX 創 60 日新高 且 量 > 60日均 × 1.5
2 權值股獨強 Top100 中 ≥ 3000 億的佔比 ≥ 50%
3 OTC 轉弱 OTC 站MA20 - TWSE 站MA20 < -5%
4 主流退潮 M4 fading 檔數 ≥ 30%
5 Breadth 下降 M2 breadth_score < 50
6 法人轉保守 M1 market_structure_score < -0.30
7 成交過度集中 M3 XQ 細產業 HHI ≥ 2500

4 級分級 → 部位上限： | 觸發數 | 等級 | 部位上限 | |---|---|---| | 0-1 | 🟢 正常 | 100% | | 2-3 | 🟡 警告區 | 75% | | 4-5 | 🟠 防守區 | 50% | | ≥ 6 | 🔴 出清區 | 25% |

市場溫度（ms_temperature.py）

8 因子加權合成（0-100）：

market_temperature =
    M2 breadth_score        × 0.20
  + M3 diffusion_score      × 0.25   ← 最重
  + M4 hotflow_strength     × 0.15
  + M5 institutional_flow   × 0.15
  + amount_expansion        × 0.10   (尚未完整實作，用 50 中性)
  + otc_strength            × 0.10
  + volatility (VIX 反向)    × 0.05

5 級分類： | 範圍 | 標籤 | 操作建議 | |---|---|---| | 0-20 | ❄ 冰冷 | 空頭防守（現金為王） | | 20-40 | 🌤 回暖 | 初升段（分批進場） | | 40-60 | ♨ 溫熱 | 健康多頭（標準部位） | | 60-80 | 🔥 過熱 | 高檔震盪（獲利減碼） | | 80-100 | 🌋 沸騰 | 末升段風險（準備出場） |

RR 環境分（同檔 ms_temperature.py）

6 條件加總（0-100）：

+20  if OTC 強於 TWSE（otc_strength_diff > 0）
+25  if M3 diffusion_state == "expanding"
+15  if Top100 換手率 > 15%
+15  if Top100 換手率 > 20%
+15  if Breadth above_ma20 在 [0.40, 0.80] 範圍（權值穩定）
+10  if Top100 中型以下 > 30%

3 級分類： - ≥ 70：✅ RR 黃金期（建議積極 RR 操作） - 40-70：⚠ RR 中性（標準部位） - < 40：🛑 RR 不利（暫停 dynamic_rr 新進場）

否決條件（失敗處理）

情境 行為
sector_map.json 缺漏 跳過 M3 / M4 lifecycle 中的同產業判斷；其他模組正常
hotflow.json 缺漏 M4 lifecycle 與 M6 訊號 4 跳過
任一子模組 raise orchestrator log warning，繼續下一模組
dashboard 任一輸入缺 顯示「—」並標 muted 樣式

不會做的事（明確界線）

  • 不做個股買賣決策（那是 RR / RRR / Dynamic RR 的工作）
  • 不取代既有 risk_control.py（仍由它做個股層級風控）
  • 不影響原本 market_structure 主頁的渲染（並列存在，不取代）
  • 不直接修改 hotflow 主要邏輯（lifecycle 是疊加的注解，不改評分）
  • 不爬新資料源（純粹合成既有資料）
