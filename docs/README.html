一句話描述

每天 18:00（台股收盤 + 30 分鐘）自動跑： 抓資料 → 評估市場 → 偵測主流 → 找配對 → 風控 → 產 HTML 報告 → 部署 GitHub Pages → 推 Email + Telegram。

系統流程（10 個 Agent 順序執行）

 1. Data Agent           增量抓 yfinance + TWSE/TPEX + TAIFEX
 2. Market Structure     大盤狀態：bullish / warning / defensive / exit
                         +60 日 Z-score 模型：foreign_pressure / etf_support / regime
 3. HotFlow              全市場 ~2,700 檔 → A/B/C 分類
 4. N-Pattern            波段結構候選股
 5a. RR Pair             簡版相對輪動（RS_MA20 穿越）
 5b. RRR Pair            嚴格版共振輪動（Z60 折返 + cycle + 雙邊極端）
 6. Chip Filter          外資/投信/自營籌碼濾網
 7. Risk Control         動作放行決策（部位 25/50/75/100%）
 8. Report Generator     HTML/JSON 子頁
 9. Custom Pairs         自選配對表（配對表.xlsx）+ pair_chart 互動圖表
10. Notify               Email + Telegram

主頁 index.html 把 1~9 的成果整合成一頁。

設計原則

  1. 單一事實來源：股票價量、法人、期貨等全部存中央 C:\CludeHome\stockHisData
  2. 每日固定流程：排程器 → orchestrator → 11 個 step → 部署 + 通知
  3. 報告固定檔名：書籤友善，舊版進入 backup/{date}/ 保留 7 日
  4. 訊號層次清楚：技術面找機會，籌碼濾，風控守
  5. 配置與資料分離：xlsx 可由使用者改、程式不動

排除問題

  • 報告沒更新：檢查 Get-ScheduledTaskInfo MS_N_RRR_Daily，或手動跑 python -m src.orchestrator
  • Telegram 失效：見 OPERATIONS.md § Telegram 重設
  • 某配對 chart 沒產出：檢查 data/output/custom_pairs.json 的 chart_status；通常是中央 stockHisData 缺該檔個股資料
  • 想加新 Agent：參考 INTEGRATIONS.md § 新增 Agent 流程
