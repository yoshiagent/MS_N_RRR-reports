⚠ 便利貼覆蓋率現況：19/22 檔有他 Agent 便利貼（多數處置股是小型投機股、不在各 feature Agent 主 universe ~664 內 → 只有處置三窗便利貼）。這正是 Tier-2 要補的覆蓋缺口：屆時可選擇性把處置股擴進指定 Agent（如籌碼/箱型）的 universe，讓便利貼貼滿。
🧠 便利貼牆怎麼運作 / Tier-1 → Tier-2 計畫
你的願景：一檔股票進候選池 = 身上貼滿便利貼（各 Agent 貼），最後經理人挑、也能做後續分析。本池用 pull 模型實現：成員（處置中 4 碼股）逐檔去讀它在各 Agent 既有 FS snapshot 的評分，組成便利貼牆。FS 快照是 (代號,日期) 雙鍵 → 便利貼有時間戳，pool_history 每日 snapshot 把便利貼隨時間留痕（journey 永久），日後做「出池後實績 ⋈ 進池當時便利貼」分析時無未來函數。

Tier-1（現在·本頁）：純觀察/隔離，不進 universe_selector、不接競標 → 零決策風險。即時把每檔處置股貼滿可得的便利貼 + 進出留痕。
Tier-2（資料累積後）：處置股價量籌碼觀察 評估「處置後實績」是否被某些便利貼組合預測（如處置中收紅+法人淨買→處置後強）。有 edge 才把合格子集接 universe_selector / 給經理人 bid（regime-aware）；沒 edge 就維持觀察/風控排除。

可轉債（>4 碼）：本身不進池當標的（無價量料），但去尾碼對應回母股，母股的 CB 也同遭處置 → 貼 🔗 便利貼（主力玩 CB+正股套利的加強佐證）。
R6 clean：純讀 disposition + disposition_study + 各 Agent snapshot，不重算。
