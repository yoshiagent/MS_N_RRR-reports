配對輪動換股決策模型 演化（顧問討論 Claude×codex×Gemini）

顧問討論紀實 ｜ Claude×codex×Gemini ｜ Z20 vs RVGOZ20 打架 → v1 兩層模型 → v1.1 2D方向票+旋轉層 ｜ 🟢 v1 上線 · v1.1 規劃
狀態　v1 兩層模型已上線 SHADOW（換股決策摘要主讀數）；門檻待回測校準、不接單。執行層護欄上線接單才做。
本頁記錄配對輪動『換股決策摘要』模型的整段演化：從 CIO 一個疑問(Z20 與 RVGOZ20 方向打架)，經 Claude 分析 + codex 三輪 + Gemini 兩輪的辯論，收斂成 v1 兩層模型。對應頁面：配對輪動精簡版(pair_rotation_lite) 的最上方橫幅。
🔗 對應上線頁
v1 讀數就在配對輪動精簡版最上方：🔁 南電/欣興 配對輪動(精簡)（🧱 B15 換股決策摘要 · 依 B14 換股時機×RVGO）。
🧭 演化時間線（從一個疑問到 v1 模型）
① 起點：Z20 與 RVGOZ20 打架
CIO 觀察 B14 主圖：Z20(ln A/B 價差) 最近說『A→B』(南電漲多)，但 RVGOZ20(資金重心) 像說『B→A』——兩個都在極端卻方向相反，到底聽誰？該如何決定最佳換股時機？
② v0：四項平權『轉折一致度』(codex×2 + Gemini×1)
第一版把 價(Z20折返)/資(RVGO)/速(Z20速度)/勢(slope) 做成 N/4 一致度，叫『轉折一致度』不叫信心分數(是執行閘門非預測)。Gemini 點出共線性：RVGO/vel/slope 全是價量家族，四項並非真獨立。
③ 拉進 RR 籌碼流向 CAB 當『獨立第三源』
CIO 問頁面下方那張 RR 籌碼流向輪動圖該不該加入。發現其 Y 軸 CAB=Qpower_A−Qpower_B (分點主力淨力) 是唯一非價量家族的獨立資料源 → 拉進來做交叉驗證(價 Z20·資 RVGO·籌 CAB)。
④ 架構辯論：誰當 Z20 的共同主角？
CIO 追問 Z20×CAB＋RVGO 輔 vs Z20×RVGO＋CAB 輔。結論：主框該用『最獨立的兩個訊號』(Z20 價 × CAB 籌)，RVGO 與 Z20 半相關 → 退成輔助；把相關訊號當共同主角＝假信心。
⑤ 收斂提案：Z20 折返當閘門 → 再看 CAB＋RVGO 是否同向
CIO 提『Z20 極端折返=閘門，再看 CAB＋RVGO 同向』。Claude 贊同並補兩但書：(1) CAB 才是獨立票、權重高；RVGO 改管動能(velocity/slope)不重複投方向。(2) 硬閘門=純右側，會漏籌碼領先的左側 → 留早期觀察旁路。
⑥ 顧問兩輪補洞 → 定案 v1
codex 支持並加 Hard/Soft 雙門檻、RVGO 用 velocity/slope 投動能、CAB 中性明確降級、旁路只當雷達。Gemini 再補：協整鎖(corr/beta 破裂 Gate 上鎖)、CAB 可用性門檻(無效🈳非中性)、流動性護欄、交易成本/time-out。收斂成 v1 兩層模型並上線 SHADOW。
⑦ 方向票 2D 化：CAB 純量 → NBC×SBD 規則型狀態分類
CIO 質疑『方向票只用 CAB 純量、應二維考量』。CAB 是雙重壓扁純量，壓掉吸籌/派發不對稱、集中度品質、哪軸在推。收斂：CAB 降級 legacy、不單獨投票；方向票改規則型狀態分類(不加權重)——NBC 定方向、SBD 定品質(散戶擴散降級·極端假突破 veto)、Q1+Q3 活躍用滾動百分位(自身∪同業)、激戰(斜率走平才 veto)。1D→2D 最大值：外資過帳/借券使 NBC 失真時，SBD 未對應集中 → 自動判低品質過濾。
⑧ 度量選型：Q1−Q3 對角 vs 角速度
在 PAB×CAB 平面加度量抓輪動。裁決：主選 B 角速度(順時針旋轉=timing/轉折)、輔選 A 對角 Qpower(低頻 regime 確認)。A 單用與 Z20+CAB double-count；B 才 additive(path/加速度)。20 天不能裸做(軸尺度錯配·CAB 凍住→假 2D)→ CAB_fast=CAB20−CAB60＋雙軸 robust-z120。統計薄→重平滑、少參數、逆時針=假突破 veto、產業脫鉤(r>2.5 卡 15 天)=配對破裂止損。
🎯 定案 v1：Z20 Gate + CAB Direction + RVGO Momentum
Layer 1 · 閘門 Gate：Z20 極端+離帶折返 才開窗+定方向；前置協整鎖(corr60<0.35 → LOCK，不論 Z20 多極端都不開)。Hard |Z20peak|≥1.5／Soft 1.2~1.5(降一級·只試單不反向換)。Z20 只當觸發器,不平權。
Layer 2 · 確認 Confirm（兩票）：CAB(=Qpower_A−Qpower_B 分點主力淨力·唯一非價量家族的真獨立源) 投方向票(換 B 則 CAB<0；|分點淨力| 不足 → 判「無效🈳」非「中性」,無效時禁試單)；RVGO(velocity/slope) 投動能票(轉向即可·不強求 level 翻負,因 level 是狀態、slope 是變化)。CAB 管方向、RVGO 管動能,分工避免與 Z20 共線。
Verdict（CAB 是唯一真確認）：CAB同向+RVGO同向=換｜CAB同向+動能中性=試單｜CAB中性=最多試單(標非雙源確認)｜CAB背離即使 RVGO同向也只能等｜雙背離=別換(接刀)。旁路：Gate 未開但 CAB 強同向 → 早期觀察(雷達·不進攻)。
執行層護欄（換股最小利差>3%才全換／Soft 只減倉／5 日 time-out 拆解／流動性>0.8×5MA／CAB 混投信外資×0.6 防借券騙線）＝上線接單才需要，v1 SHADOW 先不做；CAB 純度 v1 維持純分點＋可用性門檻。

📜 顧問原文（存證）
📜 codex 顧問全文（GPT-5.5·閘門模型收斂輪·原文）
可以收斂成 CIO 的兩層模型，而且我會明確支持：Z20 降級為 Gate，不再與 CAB/RVGO 平權。這比前一版「四項平權」乾淨很多，因為它把問題拆成：
1. 價格是否已經到可換股區？
2. 這次折返是否有資金與動能確認？
這兩題不應該混在同一層投票。

1. Z20 當 Gate：同意，但門檻要分成硬門與軟門
|Z20| >= 1.5 + 離帶折返 適合當主 Gate，不算太嚴。因為配對輪動不是每天都該換，Gate 的功能本來就是限制交易頻率，避免在中段震盪時被 CAB/RVGO 的雜訊拉著跑。
但我不建議只有單一硬門，可以分兩級：
- Hard Gate：|Z20| >= 1.5 且出現折返 → 可進入正式換股決策。
- Soft Gate：1.2 <= |Z20| < 1.5 且出現折返 → 只允許「試單」或「觀察升級」，不直接滿倉換。
如果回測後訊號太少，再把主門檻從 1.5 降到 1.3，但不要一開始就降。先保留乾淨訊號。

2. CAB 管方向、RVGO 管動能：同意，但 RVGO 不應完全失去方向資訊
你的但書是對的：RVGO 和 Z20 半相關，所以如果 RVGO 只是說「A/B 也在同方向」，它很容易只是重複價格相對強弱，造成假獨立。
最乾淨的分工是：CAB = 獨立資金方向確認；RVGO = 價量重心是否真的轉向，主看 velocity / slope。
但我不會讓 RVGO 完全不投方向票，更精確地說：CAB 投方向票、RVGO 投動能票、RVGO 的 level 方向只作為輔助不作為主確認。
如果 RVGO level 還在正值、但 slope 已經轉負，這不一定是背離；反而常是合理的早期轉折。因為 level 是狀態，slope/velocity 是變化。所以 RVGO 的判定應該避免寫成「RVGO 必須已經小於 0」。

3. CAB 常中性時，不能假裝雙源確認
CAB 中性時，模型應該明確降級，不應說「雙源同向」。CAB 同向才有真正確認；CAB 中性最多只能支持試單；CAB 背離時，RVGO 同向也不能壓過 CAB。因為 CAB 是唯一真正不同資料源，它缺席時模型信心要降級。

4. 早期觀察旁路值得留，但不能變成交易旁門
旁路值得留，因為它捕捉的是「籌碼與價量重心先轉，價格還沒進極端區」的左側訊號，對研究系統很有價值。但對實務交易（尤其非機構），旁路不應直接觸發正式換股，只能：標記 early watch、提前提高監控頻率、當 Z20 之後進入 Gate 時提高確認權重。純右側對散戶更安全；旁路適合當「雷達」，不適合當「扳機」。

5. 最終定案命名：Z20 Gate + CAB Direction + RVGO Momentum
這個版本比「轉折一致度四項平權」更好，因為它不再把觸發、方向、動能、獨立驗證混成同一個分數。CAB 中性時明確降級，RVGO 避免和 Z20 重複投票，旁路只做早期雷達。邏輯乾淨，也比較能直接落地回測。

這套『Z20 Gate + CAB Direction + RVGO Momentum』兩層模型邏輯極佳，成功解決了傳統配對交易「過早摸底」與「指標多重共線性」的痛點。我高度同意此框架，但實戰上必須針對以下 5 點進行「容錯性」的批判與修補：

① 門檻盲點：配對協整破裂 (Cointegration Breakdown) — 批判同意
ABF 雙雄雖同屬載板，但常因單一客戶比重或製程良率出現結構性分歧，此時 Z20 會因「均值不回歸」而成為奪命索。
定案修補（前置檢查）：導入「動態 Beta / 殘差走勢」檢查（60 日滾動 ADF 或 20 日 rolling Beta 偏離度）。若 Rolling Beta 偏離其 240 日均值超過 ±2 個標準差，判定「協整暫時失效」，Gate 直接上鎖（Lock），不論 Z20 多極端都不開窗。

② CAB 假中性：退化成價量兩半相關 — 批判同意
南電/欣興是法人盤，分點主力（CAB）若無大戶表態，多數時候日淨力差確實貼近 0。若把「無信號」當作「中性/可試單」，等於讓系統裸奔於價量指標中。
定案修補（可用性門檻）：設立 CAB 活躍度門檻 Active_Ratio = |A 分點買超 − A 分點賣超| / A 成交量。若 Active_Ratio < 5% 且絕對張數未達門檻，CAB 票宣告「無效 (NaN)」而非「中性」。當 CAB 為 NaN 時，全面禁止試單。唯有 CAB 活躍且投同向票，方可啟動交易。

③ 量能/參與度：是否漏了第三維度？ — 半反駁
RVGO 已含 ATR（與量能正相關），若再塞入純成交量易造成指標過載。但配對交易最怕「無量乾拉」導致假突破。
定案修補：無需新增第三維度，但限制「換股信號當日，目標股成交量必須 > 5 日均量 0.8 倍」，確保換過去的股票有流動性承接，避免流動性折價。

⑤ 最大實戰盲點 + 定案調整
盲點：外資與投信的「左手拉右手」與換股摩擦成本。分點主力（CAB）呈現賣超，有時其實是外資過帳或借券賣出，而非真正看空；且台灣一進一出換股成本約 0.5%~0.6%，Soft Gate 頻繁開窗會被交易成本吃光。
定案調整：
1. CAB 資料源升級：CAB = (分點主力淨力差 × 0.4) + (投信+外資買賣超差額 × 0.6)，納入真法人動向，才不易被特定分點隔日沖手法欺騙。
2. 加入「換股最小利差門檻」：非到 Hard Gate 且價差偏離預期利潤 > 3% 不輕易全額換股；Soft Gate 僅允許「減倉」，不做「反向換股」，降低摩擦成本。
3. 時效性防禦 (Time-out Stop)：換股後若 5 個交易日內 Z20 未收斂且 CAB 轉向，強制拆解配對（認賠平倉），防止協整破裂造成無底深淵。
📜 codex 顧問全文（度量選型：對角 vs 角速度·原文）
結論：兩者各司其職，但若目標是「配對輪動時機」，主指標選 B 角速度；A 只做低頻體制確認，不做 timing。

1. A vs B
A（Qpower_PABxCAB = Q1_power − Q3_power）本質是「A 體制 vs B 體制」的低頻確認，適合回答「現在偏抱 A 還是換 B」。但它丟掉 Q4/Q2，抓不到「正在轉折」——而你真正想抓的早期訊號正是 Q1→Q4（A 價強但籌碼轉弱）與 Q4→Q3（價格跟上籌碼）。B 角速度才是 rotation timing 指標。定性：B = timing/transition detector；A = regime/confirmation detector。不要用 A 取代 B。

2. 窗口：不要直接做 20 天版 2D，除非 CAB 也快化
PAB 快、CAB 內部 60 天慢。20 天窗內 CAB 近似常數 → 退化成「固定 CAB 正負 × PAB 跳動」＝重複計 PAB。務實：timing 版用短窗但 CAB 必須有快版（CAB_fast，如 20D 或 20/60 residual）；confirmation 版用 ≥60 天尊重 CAB 平滑尺度。

3. 角速度是否較耐受尺度錯配？只稍微，不免疫。必須把 PAB、CAB 各自 robust-z 標準化，且參考窗分開（PAB 60~120D、CAB 120D 或自身平滑尺度），重點是尺度標準化，不是只靠 atan2。

4. additive 還是 double-count？A 加進已有 Z20 閘門+CAB 方向票多半 double-count（重複 PAB/CAB 正負與象限停留），除非用作累積體制強度。B 較 additive，提供 path/transition/acceleration（一階動態），但仍由 PAB/CAB 衍生、是狀態轉移特徵不是新資料源。

5. 過度工程風險不小。單一 pair、60D 平滑、完整輪動循環數很少，統計難支撐複雜模型。務實：少參數、固定窗、只判方向與狀態、不對績效最佳化、用作 veto/alert/confirmation 不單獨下單。

6. 定案：主選 B 角速度、輔選 A regime confirmation、不要只選 A。
標準化 x_t=robust_z(PAB20,120)、y_t=robust_z(CAB_fast,120)。角度 theta=atan2(y,x)；每日旋轉 dtheta=atan2(x_t*y_{t-1}−y_t*x_{t-1}, x_t*x_{t-1}+y_t*y_{t-1})，dtheta<0=順時針。rot_5=−Σdtheta(5日)、rot_10 同理，正值=順時針輪動。距離濾網 median(r)>門檻避免原點亂跳。early_switch_alert = Q4 且 rot_5>門檻 且 valid；switch_confirm = Q3 且 rot_10>門檻 且 regime_score<0。A：Q1_power/Q3_power(60D)、regime_score=Q1−Q3。裁決：做兩者但權重不同，B 主 A 輔；快版必須處理 CAB 慢軸，否則 20D 2D rotation 是假 2D。
📜 Gemini 顧問全文（度量選型補洞·原文）
同意 B 角速度為主、A 對角為輔，此設計極佳——對角線 Qpower(位置)是靜態、有『同象限但方向相反』的滯後；B(角速度 dθ)引入時間微積分，解決動態路徑。「在哪裡」不如「往哪走、走多快」重要。

2. 60 天無完整循環，角速度站得住嗎？站不住（若直接當連續訊號），必須離散化＋平滑。南欣 ABF 年化僅 2~3 次大波段，60 天內多是噪聲。修正：先過距離濾網(r>門檻)，再對 dθ 做 5~10 天 SMA 或 Kalman，只抓趨勢性轉向。

3. 非乾淨順時針怎解讀？逆時針(dθ>0)=失效/假突破(Q1 未入 Q4 反退 Q2＝籌碼虛火/價格回馬槍)→一律 veto 停換股。亂轉(原點附近 r 極小)=無配對動能→空倉/靜止。

4. CAB_fast=20D−60D 好嗎？好，但『相減(Oscillator)』優於『殘差(Residual)』——回歸殘差引入估計窗波動與 look-ahead bias。直接用 MACD 式差值 CAB_fast=CAB20−CAB60，本質是籌碼動能加速度，破除假 2D 又保持線性。

5. 門檻怎麼定？距離 r：已做 120D robust-z、標準差歸一 → 設 r>1.0（偏離均值 1 個標準差才有動能）；median(r)>0.5 太低易在原點亂觸發。角速度 rot：不定絕對速度值，改用方向持續性——累計 5 天內 4 天 dθ<0 且累計弧度 Σdθ<−0.5(約 −30 度)。

6. 最大盲點：產業結構性脫鉤（Beta/Fundamental Decoupling）。南電偏 CPU/聯發科、欣興 NVDA 比重高，非對稱衝擊(如 AI 爆發)會永久性發散，軌跡卡 Q3/Q4 極遠(r≫3)不回頭，模型誤判『超跌/準備輪動』瘋狂接刀或不肯止損 → 毀滅性虧損。防禦：時空雙重止損——r>2.5 且維持 >15 交易日未旋轉(角速度停滯)=配對關係破裂，強制斬倉；Robust-Z 滾動窗固定 120 天，嚴禁為績效 optimize 窗長。
