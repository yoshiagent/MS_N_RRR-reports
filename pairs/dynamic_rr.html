動態 RSR — 策略與用法
⚙ 動態化 RSR vs 既有 RSR
既有 RSR/RRR：固定 Z 門檻、一次性換股
動態 RSR：依配對自身波動自動調整門檻、分批換股（30/40/30%）、有冷卻期
📊 VolRatio 波動分級
VolRatio = STD20(RR) / STD60(RR)
低波動 < 0.8 → Entry ±1.5 / Pullback 0.3 / Cooldown 3d
正常波動 0.8-1.2 → Entry ±2.0 / 0.4 / 5d
高波動 1.2-1.6 → Entry ±2.5 / 0.5 / 8d
極端波動 ≥ 1.6 → Entry ±3.0 / 0.7 / 12d
📈 3 段分批換股
Step 1（30%）：Z60 達 ±Entry_Z → 首批
Step 2（40%）：Z 從極值折返 ≥ Pullback_Z
Step 3（30%）：目標股站上週MA20 且 MA20 ↑

累計 100% 後維持持倉，等下次反向訊號。
❄ 冷卻期
每次換股後依當前 vol_state 進入冷卻：
低 3d / 正常 5d / 高 8d / 極端 12d

冷卻期內不允許反向換股，避免在 Z 大幅震盪時頻繁進出。
🛡 風控閘
市場層級：大盤 defensive / exit → 停止新增換股
配對層級：A 或 B 任一檔 週MA20 轉下 → 停止該配對

觸發時 card 紅框，警示訊息顯示在進度條下方。
🗄 跨日狀態 (state machine)
狀態持久化於 data/processed/dynamic_rr_state.json：
position / tranche_progress / cooldown_until / last_extreme_z

每日 orchestrator 跑完自動更新；
不要手動編輯 state 檔，會造成計時錯亂。
