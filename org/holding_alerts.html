🧠 監控層怎麼運作（閉環）
買→持→監控→賣 的迴圈:
研究部(各 score/警示 agent → FS) → 經理讀 FS bid 選股 → decision 持倉 → 本頁(監控層)把每位經理的持股 ⋈ 各警示來源 → 統一燈號 → 寫回 FS → 經理下一期競標時，對亮紅燈的持股自動降 bid → decision 減碼/換出。

⛔ 硬停損(任一觸發=強制換出，不需多訊號共振):
　① 成本停損:跌破平均成本 ≥ 10.0%(paper_portfolio 真實 avg_cost)
　② 移動停損:自進場後最高收盤回落 ≥ 15.0%(鎖獲利)
　③ 結構停損:跌破最新有效 SL(box_structure stage_4)
　④ 週線停損:買進後現獲利 ≥ 20.0% 時(鎖利波段續抱)，週收盤跌破前一週最低點 → 賣出(以週線判斷，避免日內雜訊)
　⑤ 爆量停損:跌破最近一根爆大量 K(量 ≥ 2.5×20日均量)的最低點
硬停損觸發 → 強制紅燈 + 扣滿 bid → 該持股下期被換出。

單一調校點:嚴重度模型、停損門檻、動作對應、扣 bid 全在 holding_alerts agent，以後只要優化這一支即可。
燈號需多訊號共振:單一輕警示(如僅結構劣化)不足以亮燈;兩個以上來源同時警示才轉黃/紅(避免單一指標誤殺)。
